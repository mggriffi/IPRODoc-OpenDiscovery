InstallerPro FAQ

I'm unable to log in to InstallerPro, even though my username and password are both correct. What's happening?

Double-check that your email and password have been typed out correctly. If you include any additional spaces after your email, the log in will fail.

 

When configuring the IPRO components for installation, I noticed a field that asks for "The name of the ADD website in IIS." Is it necessary to list the name of the ADD website as it appears in IIS, or can I leave this field blank?

If ADD Web already exists on your machine and you are upgrading to the latest version, it's necessary to input a value for this field. Open the Internet Information Services Manager on your machine and locate the name of the ADD website as it appears there.

If you do not already have ADD () installed on your machine, leave this field blank.

 

Is it necessary to install the IPRO core components in a particular order, or can I install the components in any random order?

For the services to function properly, some components must be installed before others. Both Redis Server and Identity Server must be installed before . Furthermore, must be installed before Platform Web, Platform Web API, and the Job Manager Agent. You may install Redis before Identity Server, or Identity Server before Redis. One possible installation order would be as follows:

  1. Install Redis Server

  2. Install Identity Server

  3. Install (Note: The application is not available for installation through InstallerPro. The latest version of may be found and downloaded from https://myipro.iprotech.com.)

  4. Install Platform Web

  5. Install Platform Web API

  6. Install Job Manager Agent

 

When I attempt to install Redis, I get a message saying "That port is already in use on this machine." What do I do?

Double-check that Redis has not already been installed on your machine. Open your control panel > Programs > Programs and Features to see if Redis server is listed there. If the port is in use, most likely Redis has already been installed on your machine and you can move on to the next component.

 

I'm able to install Redis Server on my machine, but unable to install any of the other components. What's happening?

There may be a number of reasons for why Identity Server and the other components are not installing properly. Ensure that all required fields have been configured with the proper values. Furthermore, verify that the Authorization Service has been installed on the machine. If Authorization has not been installed, you will not be able to install Identity Server.

 

While installing one of the IPRO components, a message appeared saying the "Install Pipeline Finished," however the progress bar still displays on the screen, along with the message, "Installing. Please wait..." Now, nothing's happening. What should I do?

An unexpected error likely occurred during the installation process. Close the application and sign back in. Either the platform will show the component as having been installed, or you will have to run the installation a second time.

 

On the InstallShield Wizard, while configuring settings for , I'm asked to input a Redis, Identity, and Platform Web API URL. What happens if I leave this section blank?

If these fields are left blank or otherwise populated with the wrong values, the application will not function properly when installed. Make sure to provide the correct values for these fields. The installer will not validate these values to confirm that they are correct prior to installation. Note that the application will still be installed successfully onto the machine, however it will not function correctly. Only by uninstalling the product, then reinstalling with the correct values will work as intended.
