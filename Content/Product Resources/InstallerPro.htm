InstallerPro Installation Guide

InstallerPro is an product used to install the core platform components. These components are required for the proper use of the application. InstallerPro may be downloaded and installed from https://myipro.iprotech.com. Through InstallerPro, you can:

  • Install core components onto a local computer
  • Create installation packages to be run on other computers
  • Upgrade installed components
  • Patch installed components

Install Prerequisites for Platform

Before you download the InstallerPro platform, ensure that your machine is configured with the necessary software. If your machine is not properly configured, you will not be able to install the needed components from the platform.

  • Ensure SQL Server is installed and available
  • Ensure Internet Information Services (IIS) Manager is available
  • Ensure ASP .NET Core 2.1 is installed
  • Ensure ASP.NET Core/.NET Core Runtime & Hosting Bundle is installed, and ensure that it has been installed after IIS has been installed, or the required features will not be installed.
  • Ensure that Authorization 2017.3.4 is installed. The Authorization Service can be found and downloaded from https://myipro.iprotech.com.
  • Ensure all users have modify access to C:\ProgramData\IPRO Tech\logs

Install Prerequisites for InstallerPro

  • Ensure all users have modify access to C:\ProgramData\IPRO Tech\logs
  • User running InstallerPro must have Local Windows Administrator (Run as Administrator) rights

Choose a Save Location

The Save Location is the place on your machine or network where InstallerPro settings and data will be stored, and where packaged installers will be placed.

  1. Open InstallerPro and log in with myIPRO credentials.

  2. The Installation / Product Selection page opens by default. Open the Settings page by hovering over the gear icon in the left navigation panel.

    The left pane expands to show the Installation and Settings tabs. Click Settings.

  3. The Save Location field displays the default path where InstallerPro settings and data will be stored, and where packaged installers will be placed.

    To change the path from the default location, click Browse and navigate to the new location you would like to use instead. When you have selected the appropriate location, click Select Folder to populate the field with the new path.

  4. If you have changed the path from the default location, select Save in the bottom-right corner to save the new location. If, prior to saving, you wish to revert the new path back to the original, click Revert in the bottom-left corner.

    Note: Once you have saved a new path, you will not be able to revert back to the default location by means of the Revert button. To use the default path as your Save Location, you will need to manually browse out to it.

     

    If you have logged in already before changing the Save Location, you will have to re-open the app to see new Save Location config values shown.

Install Components on Your Computer

Perform the following steps to install the core platform components onto your computer. For information on setting up the installation of each component, see Configure Platform Components, further down in this topic.

Note: To install the required features from InstallerPro, ensure that your computer is configured with the proper software. See Install Prerequisites for Platform to verify that your computer is properly configured.
  1. Open InstallerPro and log in with myIPRO credentials.

  2. The Installation / Product Selection page opens by default.

    Select required components for installation by selecting the check box next to each name.

    The components available for installation include:

    1. Redis Server - a remote dictionary service for platform components.
    2. Identity Server - central Identity server for components.
    3. Platform Web API - an API website component for managing the Job Manager.
    4. Platform Web - a website component that contains the Job Manager.
    5. Job Manager Agent - an Agent service that acts as a processing worker for Mass Action Print, Mass Action OCR, and Imaging jobs.

    Note: By clicking the icon beside a component, a pane opens on the right side of the screen with additional information about the component and its installation status.

     

    If there are no components listed, this means that you are not licensed for the missing components and that your myIPRO account does not have licenses.
  3. When all required components have been selected, click Next in the bottom-right corner of the screen. See Configure Platform Components for information on how to configure the components properly prior to installing them.
  4. Select View EULA to open and review the End User License Agreement. To accept the EULA and move to the next page, click Accept in the bottom-right corner of the screen.
  5. Provide configuration values for the components being installed. Fields that require a value are indicated by an asterisk. Once required values have been added, click Next to proceed through each configuration screen. Configuration values are validated to determine if they meet requirements.

  6. Once all configuration values have been added and you've clicked Next, the installation process begins. During installation, progress is reported to the user.

  7. After installation, results are provided to the user (Complete or Failed with error).

Create Packages to be Installed on Other Computers

If you are required to install components on computers with no internet access, you can create packages that get stored in your Save Location. To learn how to create these packages, review the following steps:

  1. Open InstallerPro and log in with myIPRO credentials.

  2. Select Build install as a shared package? at the bottom of the page.

    Before proceeding with the next steps, ensure the path for your Save Location is correct. When the packages are created, they'll be saved to this location. See Choose a Save Location for instructions on filling in this path.

  3. Select components for packaging. Examples:

    • Redis Server - a remote dictionary service for platform components.
    • Identity Server - central Identity server for components.
    • Platform Web API - new API website component for managing Job Manager.
    • Platform Web - new website component that contains Job Manager.
    • Job Manager Agent - new Agent service that acts as a processing worker for Mass Action Print, Mass Action OCR, and Imaging jobs.

  4. When all required components have been selected, click Next in the bottom-right corner of the screen. See Configure Platform Components for information on how to configure the components properly prior to installing them.

  5. Select View EULA to open and review the End User License Agreement. To accept the EULA and move to the next page, click Accept in the bottom-right corner of the screen.
  6. Provide configuration values for the components being installed. Fields that require a value are indicated by an asterisk. Once required values have been added, click Next to proceed through each configuration screen. Configuration values are validated to determine if they meet requirements.
  7. Once all configuration values have been added and you've clicked Next, the packaging process begins. Packages are placed into the Save Location > Packages folder, by component.
  8. Copy the package ZIP file(s) to the machine where they will be run.
  9. UnZIP contents from ZIP file(s).
  10. Open a Windows PowerShell Window as Administrator.

  11. Use the cd "FilePath" command, then run the install.ps1 by using .\install.ps1"install.ps1" file for each package into the PowerShell window

Generate PFX Files for Certificate Fields

For HTTPS installations, a server certificate must be generated and present on disk before installation. The following steps describe the process of generating the certificate--the path, friendly name, and password of which will be used when configuring components for installation.

Note: While the following instructions explain how to generate a PFX file, you could obtain one from your domain authority, such as GoDaddy. Or, you could export an existing cert.

 

InstallerPro also supports using an existing SSL certificate by simply typing the friendly name into the "Certificate Friendly Name" field. This will look up your installed certificate.
  1. On a Windows machine with Internet Information Services (IIS) installed, access the IIS Manager (can be typed in search bar).
  2. In IIS Manager, click the server in the left-hand tree.

  3. Double-click Server Certificates in the tile view.

  4. On the right side, click Create Self-Signed Certificate.

  5. Input a friendly name for the certificate and choose Personal for the certificate store.
  6. Click OK.
  7. Select the new self-signed certificate in the Server Certificates list.
  8. Click Export on the right-hand side.
  9. Complete the Export Certificate dialog box. Select an export location, enter a password and confirm the password.
  10. Click OK.

How to Fully Uninstall InstallerPro

  1. Open the Control Manager on your machine and go to Programs & Features (or Apps & Features).
  2. Find InstallerPro, right-click, and select Uninstall.
  3. Follow prompts to uninstall.
  4. Go to the Save Location specified above, in the section Choose a Save Location.
  5. Delete contents of the folder (InstallerPro.db, ZIPs, Packages).
