What's New in Enterprise Review

The 2019.6.1 focused on allowing document review to happen dynamically in both batches and searches. This functionality is only available in the Review hexagon. Some of the key features added include:

  • Tags in the Grid – When in the document grid, users can now add any tags they have access to by adding a new tag group column. This allows to reviewers to quickly compare how documents are tagged across a document set. Any tag that is selected will show in the document grid separated by semicolons. Tags can be changed using the mass tag functionality.

    A tag pallet and coding form were added in the search area of the application. An option was added to display the review grid when a user enters the case via a review pass. Users with access to review passes can also use coding forms to set a default grid view that reflects the order of the coding form.

  • Clickable Relationships/Document Permissions – Relationships now have a clickable BEGDOC column that allows users to view a document that is included in the relationships. In search mode, any document included in a relationship can be clicked in order to view it. In review pass mode, documents outside the review pass will appear greyed out with a message that the document is outside the review pass.
  • Assign a Document to a Review Pass via a Mass Action – The workflow for adding documents to a review pass were simplified. The application now has a mass action to add documents to review passes in the mass actions drop-down from the document grid. This mass action is controlled by the review pass permission. Documents added via the mass action are automatically batched according to the chosen review pass.
  • Import From Active Directory/CSV - New users can be imported from your active directory account or a .CSV file.
  • Improved User Management - You can now view and “manage”, which groups a user has been assigned to from the user's page. Additionally, you can specify which IP addresses have access to your application and force users to reset their password on next login.
