Anti-Virus Exclusions - (2021.9.0)

Generally, IPRO recommends not having anti-virus active monitoring on the machines with IPRO components on them. This especially applies to workers because it can cause task timeouts, slow down discovery and processing, or even prevent document access. Instead IPRO recommends to scan incoming data before ingesting it. If anti-virus is mandatory, below is a list of directories, executables and services we recommend making exceptions or exclusions for. Please keep in mind this isn't an exhaustive list, and will need to be updated with each new release.

Also see: 2021.9.0 Network Ports Requirements.

Virus Scanning Best Practices for eDiscovery

Maintain network compartmentalization

  • Service accounts should have minimal network permissions

  • Restrict outbound network traffic from processing networks

Observe general best practices

  • SQL Server: http://support.microsoft.com/kb/309422

  • Hyper-V Hosts

    • If you are running antivirus software on the physical server, consider excluding the vmms.exe and vmswp.exe processes. Also, exclude the directories that contain the virtual machine configuration files and virtual hard disks from active scanning. An added benefit of using pass-through disks in your virtual machines is that you can use the antivirus software running on the physical server to protect that virtual machine.

    • Windows Server specific http://support2.microsoft.com/kb/822158

  • Exclude antivirus product install directories (engine specific)

  • Evaluate the priority of performance over AV aggression, the better performing settings are going to be the ones that minimize scanning of in-process data.

  • Exclude Service account Temp Folders

  • Exclude %PROGRAMFILES% (AGGRESSIVE)

  • Exclude %PROGRAMDATA% (AGGRESSIVE)

  • Exclude %USERPROFILE% for service account

Network Share Exclusions

  • Exclude file shares that house Customer original Data.

  • Exclude file shares that house intermediate processing (eCapture Client directories).

  • Exclude file shares that hold exported results.

  • Exclude Eclipse Case File paths, including native & image paths.

Programs to exclude for Ipro troubleshooting on all machines:

  • Powershell, allows us to troubleshoot a myriad of issues

  • Command prompt, same as powershell

  • Fiddler, a web packet sniffer that helps read packets sent between components
