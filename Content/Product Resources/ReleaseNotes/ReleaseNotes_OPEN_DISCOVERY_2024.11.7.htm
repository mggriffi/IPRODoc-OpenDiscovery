Key Features

Review

  • Mass Action Download Performance Enhancements

    Speeds have been significantly improved over previous and original implementations.

  • Create Fields on the Fly During Import into Eclipse Admin

    You now have the ability to create new fields on the fly when using Eclipse Admin to load data imports. A new option is available when choosing the fields '<Create New Field>' which when used allows configuration of a new field in real time. This prevents the need to have to stop the import workflow and exit in order to create a new field reducing time spent mapping fields.

  • Mass Field Creation in Eclipse Admin

    Multiple fields can now be created at a time using the 'Import CSV' button and supplying a CSV that is pre-populated with field information allowing increased efficiency when generating a large list of fields.

  • Print to PDF Performance Improvements

    Print to PDF performance has been improved so reduce time to complete these requests by 20%.

  • Support of Nested Searches in CSE Templates

    Increased support of CSE templates to allow import of nested searches.

Reports

  • Additional eCapture Reports added to the Reports Module:

    The Deduplication Overlay report from the eCapture Controller is now available in Open Discovery so that this report is more easily accessible.

    The Deduplication Detailed report from the eCapture Controller is now available in Open Discovery so that this report is more easily accessible.

  • Enhanced Data Extract Errors Detailed Report in .

    A new aggregate table has been added to the Data Extract Error Detailed report so that it more closely resembles the older report available from the eCapture Controller.

 

Resolved Issues

Review

AR - 22020 - With the updated Export Sets\Production UI an issue was present causing export page counts to be calculate incorrectly. This has been updated to properly show the correct page counts.

AR - 22043 - Certain users could experience extreme slowness or failure when attempting Self Service uploads larger than 20 GB in size. Changes have been implemented to increase efficiency and ensure higher success on larger uploads.

AR - 22023 - With the release of the new Mass Action Download feature there was a notable drop in performance when generating DAT and\or CSV output. We have gone back and made some changes to drastically improve that performance and have been able to exceeds pre 2024.11.6 speeds not only for DAT and\or CSV output, but also for those that include Natives.

AR - 21921 - An issue has been resolved where Smart Folders could disappear visually in cases containing a large amount of Smart Folders.

AR - 21056 - Native file validation performed in Eclipse Admin has been improved to better ensure better consistency and accuracy when performing the actual export out of Review. Reporting has also been improved to better identify specific files which may be missing valid native path values.

AR - 20188 - Users could be kicked out of a case if they lack the 'Manage Indexes' permission when accessing Case Settings. Handling has been improved so the user maintains their logged in session as expected.

AR - 20118 - A concurrent user error could be encountered in Review when SQL time zone is different from IIS/Auth which could cause users to be kicked out of a case. Handling has been improved to prevent this error and maintain the users session within a case.

AR - 20116 - Print to PDF performance has been improved so reduce time to complete these requests by 20%.

AR - 18548 - The System Processing ID field is now read as a numeric value and sorts as expected.

AR - 18008 - Native optimization could fail when documents do have have a valid native path. Better handling has been implemented to ensure job success by skipping invalid native paths.

AR - 16858 - Increased support of CSE templates to allow import of nested searches.

AR - 15273 - Users could run into problems saving changes on documents when the case date format used the hyphen delimiter. Changes have been made to better support this format and prevent failures saving changes to documents.

AR - 12909 - Active learning indexes would fail to create when multiple instances of a tag name were present. Handling has been improved to support this scenario. The associated "IA” set of fields have also been updated to better reflect this, the naming convention will now report in the following format “{fieldName} ({reviewPassName}-{tagName})”

AR - 10804 - In certain situations users need the ability to export redacted text, but not the redacted images associated. Handling has been introduced to allow for this scenario using the 'Apply Redactions' option under 'Redaction Options' and 'Include Images' is disabled under 'Images'.

AR - 10710 - In certain situations the Ipro Analytics Service could stop with no warning leading to issues with Active Learning. Resiliency has been improved to recover and continue processing these job types.

AR - 22221 - Additional fields for configuration could be skipped when the 'Load to Relativity' option is selected during a Production\Export could result in failed output. These fields are now required to ensure they are populated prior to starting the Production\Export.

Processing

AP - 6034 - When exporting multi-page images or PDF's from Legacy eCapture Processing jobs the OCRT line of DLF load files would list the Processing Directory. Handling has been implemented to properly utilize the output directory path configured in the Export Job.

AP - 5843 - Streaming Discovery Imaging and Mass Action imaging could leave extra working data in the Temp Directories. Cleanup improvements have been made to better manage this storage and ensure these files are removed when no longer required.
