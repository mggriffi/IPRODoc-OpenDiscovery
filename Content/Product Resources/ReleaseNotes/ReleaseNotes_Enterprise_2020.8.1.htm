Batch Management

  • New Batch Management features include a "Case Header" for Case actions:

  • Below the "Case Header" is a "Review Pass Header" for Review Pass actions:

  • In the "Review Pass Header", there are two new settings tabs: Batch Management and Review Pass Settings. The Batch Management tab also includes a Completed/Not Completed batches toggle button (default is Not Completed).
  • There is a right pane that shows Batch ID, status, and additional Review Pass information and actions:

  • In this pane, the UNASSIGN button allows you to unassign a checked-in document batch. The button is disabled after unassigning the batch or viewing completed batches.
  • The "BatchIdentifierSearchItem" field is available for searching by a single Batch ID and for navigating to search with a single filtered batch.

  • In the "Case Header", there is a button, when clicked will generate a downloadable CSV report (Reviewer Batch History Detail) that contains three new columns: Reviewed, Not Reviewed, and On Hold.

    See the following example:

  • In the Batch Management tab, the Batch ID column IDs are hyperlinked. A clicked hyperlink allows the Review status of the documents within that batch to be viewed in Visual Search. A batch that contains no documents shows a document result of zero.
  • The Batch Management tab automatically displays Not Complete batches by default.
  • The batches list in the Batch Management tab automatically updates when a new batch is created.
  • Case Batch Timeout default is 1440 minutes (24 hours). The number of minutes is editable. The maximum allowed value is 43,200 minutes (30 days).

    This will now allow the user to remain assigned to the batch when entering the case.

  • Expanded user interface options to allow coding form and tag palette to be resized during review.

  • Batch Management: Gain visibility to manage review passes within an easy to use dashboard. Allows reviewer management to quickly filter down completed batches, perform validation, unassign batches, and more. See Manage Batches.

  • Batch Timeout: Customize the amount of time a reviewer can keep the same batch, default 24 hours.

  • Tags: Added the ability to right click from tag palette to create and rename tags for easy, on-the-fly tag customization. See Right-Click Options in the Document Viewer Tag Pane.
  • Performance Enhancements: Decreased loading time in the following areas up to 99%:

    • Opening a case or review pass

    • Faster rendering of native files within a review pass. See Review Pass Document Loading Acceleration.

    • Creating, refresh, updating, and deletion of Smart Folders

    • Loading and search results returned in Visual Search

    • Tag filtering, tally, and sorting in the document grid

    • Saving Keyword List results to tags and fields.

Right-Click a Tag

Within the Document Viewer, right-clicking a tag group or tag allows for the following actions.

Right-click to Add Tags

There is now the option in the Document Viewer to click a tag group and add a tag.

Right-click to Rename Tags

There is now the option in the Document Viewer to click a tag and rename it.

Performance

Performance Increases

We have made performance increase in the following areas:

  • For both the Review and Process databases, data is now pulled every 10 seconds and saved to a table. The data is then analyzed to access concurrency issues affecting end-user experience in the UI.
  • Smart Folder create, refresh, update, and delete (C.R.U.D.) time has decreased significantly.
  • Tags loading time within the Visual Search grid has decreased significantly.
  • Tag group filtering time has decreased significantly.
  • Document generation within a case has improved by about 90%.
