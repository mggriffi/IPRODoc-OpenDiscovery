What follows is a list of the new features included in the 2019.7.0 release as well as a list of resolved issues.

ECA & Review

(New, Simplified and Modern Interface)

New Features

  • Concept Wheel unified with Visual Search offers users a new way to be interactive with clusters while still having the existing benefits of filtering, reporting and search within the same interface. For more information, see Define a Visual Search.

  • Collapsible Timeline provides more real estate to the search view so that reviewers can focus on what's important when building a visual search. To learn more about the Timeline, see Define a Visual Search and Define the Timeline.

  • Find Similar documents of interest based on concepts while reviewing a document or by using advanced search. For more information, see Overview: Relationships.

  • Tags on the Grid enables reviewers to quickly filter and compare document tags across a document set. This can be powerful when combining this method while pulling similar documents based on concepts, as well as near-duplicates, email threads, and other relationships. For more information, see Work with Search Results.

  • New batch report, “Reviewer Batch History” details user activity and calculates edits per hour by a specific reviewer. See Reviewer Batch History Details Report to learn more.

Improved Features

  • Smart Folders available in a new interface that allows users to easily organize and review document sets. For more information, see Work with Smart Folders.

  • Enhanced Quick Search to provide a bread crumb display of the search history within the document grid. See Filter Search Results to learn more about quick searches.

  • Simplified new interface for Document Compare providing a side-by-side view while highlighting discrepancies between the documents within the relationships panel. For more information see Working with Relationships.

  • Coding Form will now automatically resize field section to allow users more room for displaying tags. For more information, see Manage Coding Forms.

  • Updated interface for combining search history criteria to create new searches. For more information, see Perform an Advanced Search.

Processing

New Features

  • New Job Manager Web Interface to view, manage, and prioritize the first phase of review and processing jobs. Streaming and other job types will be included in following releases. See Overview: Job Manager to learn more.

  • Real-time job status notification within the application is provided when a new job is submitted.

  • Mass Actions capabilities are now available through the Review Interface, allowing users to apply OCR, Print to PDF, or Imaging to a selected list of documents. For more information, see Perform Mass Action OCR, Perform Mass Action Print to PDF, and Perform Mass Action Imaging.

Improved Features

  • Improved Text Extraction of WordArt and SmartArt found within Microsoft Word documents to ensure text extraction.

  • Optimized identification and flagging inline images, “Is Inline Image” for image files and “Inline Images” for documents with inline images. For a full list of QC flags, see Work with the QC Flags Window.

  • Enhanced handling of OCR confidence by reporting and flagging documents with low confidence scores. In prior releases, they would be marked as failures but now they can be identified with "Low Confidence OCR" flag or by utilizing the "OCR Status" report.

  • Target a select document set for Email Threading instead of the entire case. For more information, see Configure Email Threading in IPRO Analytics.

  • QC now comes with enhanced inline image identification.

Administration

New Features

  • New Permissions to manage the new Job Manager Web Interface. See Permissions List for a full list of available permissions.

  • New Permission to manage users who have access to Mass OCR.

  • Provide an option to override group permission for a specific case. For more information, see Assign Permissions.

Improved Features

  • Simplified user interface for Security Permissions allowing for quick visibility into group permissions and cases assigned to each group.

  • New interface for re-batching unreviewed documents into new batches. See Create Review Passes and Batches to learn more.

Miscellaneous

Improved Feature

  • There is an Q compatibility upgrade to support the 2019.7.0 release.

Resolved Issues

Review

  • Checking in a batch as, “Not Completed” status would cause the batch to be temporarily unavailable but is now accessible for reassignment.

  • Timeline date filter displays placeholder text of the expected date format to be used when searching documents.

  • TIFF Images generated by third-party tools and that do not include complete resolution unit information may result in shifted redaction(s) during production.

  • Analytics could encounter an unhandled exception resulting in a “One or more errors occurred” message. Handling has been enhanced to improve efficiency and prevent errors.

  • Deleting an Analytics index would cause the associated fields to remain in the database; these fields are now removed automatically during the deletion process.

  • TAR reporting could result in blank values for expected data. Logic has been improved to ensure these are populated in report output.

  • TAR batch check-in failed causing values needed to continue TAR process not to populate.

Processing

  • Improved detection of hidden rows within Excel files.

  • Error handling during a Production has been improved to account for issues when writing to a load file.

  • Conversion of extracted text from large native TXT files was modified to convert into sections to prevent excessive use of memory.

  • Improved case templates to automatically include and combine metadata fields (Author, EntryID, Form, Lotus Views, Universal ID) even though they have not yet been extracted by the processing engine.

  • Improved identification of embedded documents, specifically for OneNote files.

  • Identification methods have been improved to consider inline images found within the body of a document.

  • Path information was being populated in the Exported Native Path export field even though no native text placeholder was created. This has been resolved by not providing this information unless an actual native text placeholder is present.

  • Identification of attachments in Structured Storage files has been updated to identify additional embedded documents that use MailAttachment streams.

  • The range for identification of a standalone ZIP file has been increased to prevent being recognized as a multi-volume segment.

  • Improved handling to identify and display attachments within the header.

  • Improved handling to identify and remove duplicate header information.

  • A re-process job performed in the QC Module would generate only load file output with UTF-8 encoding and not respect the settings of the case. This handling has been updated to match encoding based on export settings for the originating job.

  • Identification and methods for interacting with self-extracting ZIP (.exe) archives have been implemented to properly extract contents and handle appropriately as a container.

  • Additional error handling has been added to identify and flag Visio (.vsd) files with inconsistencies in extracted text. These files are marked as exceptions and can now be re-processed in the QC module. Users can now use the “Reprocess in native application” option OR “Reprocess selected in native application via ShellExecute print” when Visio is installed on the local computer.

  • A validation has been added to ensure that hidden text options set within Microsoft Word do not take precedence over the conflicting settings found within eCapture.
