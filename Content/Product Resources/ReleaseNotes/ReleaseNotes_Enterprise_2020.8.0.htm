New Features

Reinforced Security

Federated Identity Single Sign-On (SSO)

Access case documents even faster and more securely with single sign-on (SSO) support. SSO logs you into through your organization's existing identity provider or directory service. Increase organizational security and decrease administrative overhead with our integrated SSO support. One less username and password to maintain! Federated Identity supports the ability to set up one identity provider for each system environment by using the OpenID Connect standard. Third party providers that can authenticate through the OpenID standard include Google, Microsoft, Okta, Apple, and PingID.

TLS 1.2 Compliancy

Release 2020.8.0 is TLS 1.2 compliant.

Note: Using Single Sign-on (SSO) requires enabling TLS 1.2 with an outbound HTTP connection to the identity provider.

Multi-factor Authentication (MFA)

MFA, also known as Two-factor Authentication (2FA), enables you to strengthen security by requiring a time-based one-time passcode (TOTP). sent to a user's device, such as a mobile phone.

  • Enable/ or disable 2FA
  • Administrators can reset a user's 2FA if the user's phone is lost, stolen, or replaced.
    • The user may be required to re-authenticate their device and no longer have the 2FA backup key.
  • MFA status is displayed on the user’s profile.

Review

Visual Search Modernization

New Visual Search provides an approachable and interactive search experience letting you uncover critical evidence faster and in ways that you couldn't with traditional search capabilities. Interactable charts work the way you do, giving clarity to case evidence without requiring additional costly training.

  • Searches are now transferable between Visual Search and Advanced Search
  • Interactable charts offer a new level of depth to the searching experience without requiring additional training.

Note: No new charts have been added in this release.

Similar Documents

The Similar Documents Relationship has been updated to display documents with a Similarity Score of 70 or greater. The previous Minimum Similarity Score display was 10, which typically returned a higher number of similar documents.

Hot Keys

Spending too much time in review? Scale up through Hot Keys and reduce the times your hands need to leave the keyboard with efficient hot-keys mapped to the most important document review functions. Hot-keys help you increase efficiency without losing quality.

  • Tags can be assigned keyboard shortcuts in the tag palette screen.
  • A hot key can be assigned only once, then it is grayed out.
  • Numbers 0-9 can be used. Letters A-Z can be used. No more than 36 tags can get assigned shortcuts.
  • Tool tips are added for any hot key that has a keyboard shortcut in the review pass pop-out and the search pop-out.

Reporting

Streaming Discovery Detailed and Storage Reports now available from within

Two system level reports have been added to assist with billing.

  • Processing Storage Report: System Level results of processed data per date range are reported as well as current database sizes. All parameters are optional. Columns may be sorted. All sizes are exported in GB. Default granularity is set to Managing Client / Client Level.
  • Review Storage Report: Reports the total size of all records recorded in the selected Managing Client's case database(s) that were loaded within the selected date range. End Date will represent the most recent date that terminates the desired date range. Default aggregation is by Case, Client, and Managing Client. All parameters are optional.

When processing data, three frequently needed reports are now available through Reports library.

  • Discovery Detailed Error Report: This report originates from and has been added to Reports. List of errors, status messages, and password-protected documents encountered during discovery. The end of the report displays a list of password-protected files that includes the Item ID, Location, item type, file, and physical location for each file.
  • Data Extract Detailed Error Report: This report originates from and has been added to Reports. List of extraction and OCR errors and status messages encountered during data extraction. The end of the report displays a list of password-protected files that includes the Item ID, Location, item type, file, and physical location for each file.
  • Processing Storage Report: This report returns the size of all records in the associated project (case). Results calculated per date range are reported per custodian, Job type, and individual Job level. Date range is optional. Columns may be sorted. All sizes are exported in GB.

When reviewing data, an additional report is now available through Reports library.

  • Review Storage Reports: This report returns the size of all records in the associated case database that were imported within the selected date range. End Date will represent the most recent date that terminates the desired date range. All parameters are optional.
