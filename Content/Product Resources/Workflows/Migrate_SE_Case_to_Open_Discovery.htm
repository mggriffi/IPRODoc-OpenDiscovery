Migrate an Eclipse SE Case to

Migrating a case from Eclipse SE to is a multi-step process that involves migrating over both the case data and the production history.

  1. Export the case from Eclipse SE.
  2. Create a case in . This is the case into which you will be importing the Eclipse SE case data.
  3. Import the Eclipse SE case data into using the Eclipse Admin tool.
  4. Export the case production history from Eclipse SE.
  5. Import the Eclipse SE production history into using the Eclipse Admin tool.

The below diagram shows the export and import process for a given case from Eclipse SE to .

Important: There are some case elements that do not transfer from the Eclipse SE case to the case. These include:

  • Saved Searches

  • User Security

  • Transcript Issues, Notes, Linked Documents

  • Smart Folders

  • Case Documents

  • Page-Level Tags

  • Product Shield

  • Control/Pick Lists

  • Production Settings

  • Persistent Highlighting

  • Custom Layouts

  • Partial Dates-i.e.,01/00/2011

After the import of case and production history information is complete, you can set up these items, with the exception of transcripts, in the newly created case.

 

Note: Private tags also are not exported from Eclipse SE. To Convert a Private Tag into a Public Tag: To do so:

  1. Create a Public Tag Group - This can be completed in Admin or Desktop, depending on permissions.

  2. Create a tag under a public tag group.

  3. Run a search on the private tag items - This can be accomplished by a right-click context menu action, or using quick search or advanced search.

  4. Finally, apply the newly created public tag to the search results - This can be accomplished by selecting all records in the search result and then the tag check box, or using the right click context menu to "apply to search results".

Export Eclipse SE Case

  1. From Eclipse SE Administration, select Import/Export > Export Data.
  2. Select Case.

  3. Select Entire Case.

  4. On the Step 2. Select Output tab.

    1. Under Output Directory:

      • Select Output Location.

      • Check Copy Image files to output folder.

      • Check Copy Native files to output folder.

      • Check Export Text as File.

      • Check Export .CSE.

    2. Select the following Load Files: . DLF, .DAT, .OPT, .LFP.
    3. Under Items to Export check: Document Level Tags, Redactions, Production History.

    4. All Database Fields selected.

      Important: For "Image/Native File Naming Options" select Image Key.

  5. On the Step 3. Finish tab, click Start Export.

Import Eclipse SE Case Data Into OPEN DISCOVERY

Create your case in . You can use the.cse file you exported or the default case template. Make sure and add any additional fields needed before import. Copy the entire Export Volume to your Case Data location.

  1. Import Eclipse SE Case Data into using Eclipse Admin > Import > Import Data.

  2. On the tab Step1. Select Source, in the Import file field use .DLF.

  3. On the tab Step 2. Map Fields:

    1. Select Auto-Select by Field Name and Map any other fields needed.

    2. Edit/Update paths for Text and Natives.

    3. After all fields are mapped select Append all records > Import.
  4. Import image file.
    1. Select Import > Import Data.
    2. Select Source; LFP or OPT file.
    3. Edit/Update paths for images.
    4. Overlay only > Import.

Export Production History from Eclipse SE

There are two options for exporting the production history from your case.

  • Production included an Image File (LFP or OPT).

  • Production did not include an Image File.

Production With Image File

  1. Export Production History using Eclipse SE Administrator > Import/Export > Export Data.

  2. Select the case.

  3. On the Step 1: Select Case tab select Production History.

  4. On the Step 2: Select Output tab:
    1. Under Output Directory, select the Output location and check Copy Image Files to Output folder.

    2. Under the Other Load files section, check .LFP.

  5. On the Step 3. Finish tab, click Start Export.

  6. Copy your Production History to your Case Data folder in .

Production Without Image File

Option 1: Export Production History

  1. From Eclipse SE Administration select Import/Export > Export Data.

  2. On the Step 1: Select Case tab select Production History.

  3. On the Step 2: Select Output tab:

    1. Enter an Output Directory.

    2. Make sure that Copy Image Files to Output Folder is selected.

    3. Select LFP under Other Load Files.

  4. On the Step 3. Finish tab, click Start Export.

Option 2: Re-run the Production

  1. Run each Production, selecting the Production Job name from the Production History tab.

  2. Click Load Production Settings.

    1. Select/run the search originally used for the production.

    2. Click the Use Range for Production button.

  3. Select Output Options.

    1. Select Images for Output Type.

    2. Select LFP as a Load File.

  4. Select Endorsement Options.

    1. Select the Production Job name under Select Image Key > Use Produced Key.

      This will export the documents with the correct production bates numbers.

  5. Uncheck the Update Production History box and run the production.

  6. Copy your Production History to your Case Data folder in .

Create a Cross Reference File in Review

  1. Create Cross-Reference File in Review.

    1. Open the case in .

    2. Run your Production Search.

    3. Modify the Grid to show only BEGDOC and PRODBEGDOC fields.

    4. Select all documents,click Actions and then select Export.

    5. Select Output Type > DAT File.

    6. Click Export.

    This is your Cross Reference File. Copy it to your Production History folder.

  2. Edit the Production History .LFP

    1. Use Find and Replace to replace the volume.

    2. Update the Path with the full UNC path.

Import Production History into Your Case

  1. In Eclipse Admin, select Import > Import Production History.

  2. In the Cross Reference File field, click on the folder button and navigate to the location where you stored the cross reference file you just created. Use the Cross Reference File you created.
  3. In the Image Load File field, click on the folder button and navigate to the location where you stored the updated LFP file.
  4. Check the First Line is the header check box.

  5. Enter a Production Name.

  6. Click the Import Production Job button.

When the import is finished, you have successfully completed the Eclipse SE to case migration process for your case.
