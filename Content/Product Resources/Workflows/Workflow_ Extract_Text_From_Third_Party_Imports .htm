Workflow: Extract Text from Third Party Imports

When you receive third-party data that contains natives but no text, there are two different workflows you can use to extract that text. One option is to mass action image and mass action OCR the documents without text. Another option is to utilize eCapture processing. The following workflow will discuss the steps using eCapture. You will need to work in , eCapture Controller, eCapture QC and Eclipse Admin to perform these tasks.

Create the Import for Third-Party Data

Step 1:

The following steps require you to work in Case Management, the Review module and Media Manager.

  1. Create an Export of the natives in your review case that need text extracted. See Export Data for Multiple Records.

    Note: If all natives already exist in the volume provided by the 3rd party, you can use that volume for processing.

  2. Create a new processing case in . See Create a New Case in [%=Primary.Name%].

    Note: This one case can be used for all similar jobs across all matters. Jobs can be deleted after text is exported and imported to the review case.

  3. Confirm that no Import Series has been created or linked to the new processing case. For more information on Import Series, see Overview: Import Series

  4. Create a Media Manager Delivery, Media and Location based on the location of the natives. See Create New Media.

    1. Process the Media (native documents). Use Streaming so that every document has extracted text or OCR. See Process Data Locations.

      Note: Use Custodian to track sets of documents that need text.

Step 2: eCapture QC

The following steps must be completed in the eCapture QC module.

  1. In eCapture QC, open the Streaming Job you just processed.

  2. Disable ‘View Families Separately’ to allow Export Set creation.

  3. Once the job is open, right-click the session tab and select Search Through Current.

  4. Filter the Streaming Job for AbsoluteParent = 0. This will return all top level items and should equal the same number of natives originally processed. Click OK when finished.

  5. Right click the session tab and select Create Export Set.

  6. Select Create New and enter a name.

Step 3: eCapture Controller

The text export will be created from the eCapture Controller.

  1. Create an export job from the Export Set created.

    1. Select your eCapture Project

    2. Right click on Data Extract Exports

    3. Select New Data Extract Export Job

    4. Specify the Export Parameters:

      1. Select the Export Set you just created and select Direct to disk.

      2. Export Text and DAT. Include BEGDOC and TextPath.

      3. Choose ‘Export Directory Path’

      4. Update Image Key Numbering to ‘Predefined’ based on ‘Use filename’

Step 4: Eclipse Admin

Once your text file is created, you can overlay that text into the Review case. See Create the Import for Third-Party Data.

Make sure you select Overlay Only and Import Text from Path before Importing.
