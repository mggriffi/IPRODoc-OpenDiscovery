Workflow for Users and Group Management

This topic covers a standard workflow for user and group management in . User and group management takes place in the System Manager.

Open the System Manager

To open the System Manager and access the Users and/or Groups tabs, follow the steps below:

  1. Launch the application and sign in.
  2. Click the Settings icon in the top-right corner of the screen. The Settings icon is a global button that appears in every module of the platform.

  3. The System Manager opens. In the left pane of the System Manager, the Users tab opens by default.

    Use this tab to create, modify, or delete users.

  4. To manage groups, click the Groups tab in the left pane of the System Manager.

    From here you can create, modify, or delete groups, as well as assign users, permissions, and cases to groups.

User and Group Workflow

Proceed with the following sections to walk through a standard workflow for user and group management.

Create a User

  • Identify required users and gather/identify basic information (names, email addresses, passwords).
  • Add the new user by following the steps laid out in Create New User.
  • Set the user as Active (default setting) in order to allow the user access to .

Create a Group

  • Add a new group by following the steps laid out in Create New Groups.
  • Assign users to the new group.

  • Set permissions for the new group. A group's permissions filter down to the users assigned to it.

    Note: Users cannot be assigned permissions directly. You must first assign permissions to a group, then assign users to that group, for those users to receive the specified permissions.
  • Assign the group to the appropriate cases.
