Install Enterprise 2022.10.0 Review Components

The following sections contain the steps to install the components.

Install Eclipse (listed as Review in InstallerPro ) in the following order:

  1. Eclipse Service on Web Server/IIS Server

  2. Admin to any Component Server

  3. dtSearch Service to the Analytics/dtSearch Server

  4. Analytics to the Analytics/dtSearch Server

Install InstallerPro

  1. Install InstallerPro on a machine that can access myIPRO (myIPRO.iprotech.com), or if it’s already installed, launch InstallerPro.

  2. Use your myIPRO account to log in to InstallerPro. You should see the following:

  3. Select the IPRO Add package you want to install and click Next.

    • If you need to make a package installer, select the check box at the bottom left.

  4. For each component, fill out the configuration information as requested (see below for details)

  5. Once the criteria is validated, the component should install or generate the package. To execute a packaged installer, run its PowerShell file on the appropriate machine.

    • The packaged installer is saved to
      C:\Users\[USER]\AppData\Roaming\InstallerPro by default.

Eclipse Service

IIS Fields

  • Ipro Enterprise Web URL: This is the same as your complete ADD Web URL. (Example: https://addweb.company.com:1125) You do not need to include the port if the port is 443 or 80.

  • Certificate Path: This is the path to the certificate on disk. This is only required if the certificate does not have a friendly name.

  • The Certificate's Friendly Name: The Friendly Name of the certificate used by ADD Web in IIS.

  • The Certificate's Password: If the certificate is password protected, here is where you can put the password in.

SERVICE Fields

  • The Eclipse Service in IIS: The name of the Eclipse website in IIS, for example, EclipseWeb.

  • Eclipse Host Name: The Eclipse Web IIS binding hostname, for example, eclipse.iprotech.com.

  • Eclipse Service Port: The port used by the Eclipse Service. Verify this port, as it will not auto populate with your previous inputs.

  • Service Account Domain and User: The username and domain for the account that runs the service once it is installed. This account should be a domain account that has local administrator privileges.

  • Service Account Password: The password for the domain account that runs the service.

GENERAL Fields

  • ADD Web API URL: The ADD Services URL (not the ADD Web URL). You are not required to include the port if the port is 443 or 80.

  • IPRO Identity Server URL: The same as your complete ADD Web URL, for example, https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

  • Platform Web API URL: The same as your complete ADD Web URL, for example, https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

  • ElasticSearch URL: If you have Elastic Search set up, you would put the URL for Elastic Search here.

  • dtSearch Host Name: Host name of the server to which dtSearch is installed, for example, tst-suptrk054

  • dtSearch Service Port: The port used by dtSearch (5100 is the default).

  • Analytics Host Name: Host name of the server to which the Analytics service is installed, for example, tst-suptrk054

  • Analytics Service Port: The port used by the Scheduler service (10900 is the default).

  • System Events Command Server URL: The same as your complete ADD Web URL, for example, https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

  • IPRO SuperAdmin Username: This should always be administrator@iprotech.com.

  • IPRO SuperAdmin Password: The password for the administrator@iprotech.com account.

  • AllowedSites URL: This will be your ADDWeb URL. You can also use * as a wildcard.

DATABASE Fields

  • SQL Server: The full SQL Instance name where the Eclipse database is.

  • Database Name: The name of the Eclipse database.

  • SQL Username: The username of the SQL account made for the environment.

  • SQL Password: The password for the SQL account made for the environment.

  • Create new Eclipse Database (Yes/No): For new installs, choose Yes. For upgrades, you must be able to install with InstallerPro OR build/install packages before performing this step. Once the above is confirmed, choose No (use the existing database).

  • SQL Server: The SQL instance that hosts or is hosting the ADD Configuration database.

  • Database Name: The name of the ADD/ Configuration database. For new installs, use IPRO_EnterpriseConfig.

  • SQL Username: The SQL account username that ADD uses to communicate with SQL. We recommend that this NOT be the sa account.

  • SQL Password: The password for the SQL account that ADD uses to communicate with SQL.

  • Eclipse Service Port: The port number used by Eclipse Web. Verify this port, as it will not auto populate with previous inputs.

DATABASE Fields

  • SQL Server: The full SQL Instance name where the Eclipse database is.

  • Database Name: The name of the Eclipse database.

  • SQL Username: The username of the SQL account made for the environment.

  • SQL Password: The password for the SQL account made for the environment.

dtSearch Service

SERVICE Fields

  • dtSearch Host Name: Host name of the server on which dtSearch is installed.
  • dtSearch Service Port: The port number used by dtSearch. Default is 5100. You must manually set the port; InstallerPro will not carry this setting from previous items (like Eclipse Web).
  • Service Account Domain and User: The username and domain for the account that runs the service once it is installed. This account should be a domain account that has local administrator privileges.
  • Service Account Password: The password for the domain account that runs the service.

GENERAL Fields

  • ADD Web API URL: The same as your complete ADD Services URL, for example, https://addweb.company.com:1124. You are not required to include the port if the port is 443 or 80.

  • IPRO Identity Server URL: The same as your complete ADD Web URL, for example: https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

Analytics Service

SERVICE Fields

  • Analytics Service URL: The name of the server Analytics is installed to, with http:// and port number. This URL should always be http, never https. For example, http://IPROAnalytics:10900

  • Auth Server IP and Port: The IP address of the server on which Auth is installed plus the port number (port should be 9800). You must manually set the port; InstallerPro will not carry this setting from previous items (such as Eclipse Web).

  • Service Account Domain and User: The username and domain for the account that runs the service once it is installed. This account should be a domain account that has local administrator privileges.

  • Service Account Password: The password for the domain account that runs the service.

GENERAL Fields

  • IPRO Identity Server URL: The same as your complete ADD Web URL, For example, https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

  • IPRO SuperAdmin Username: This should always be administrator@iprotech.com.

  • IPRO SuperAdmin Password: The password for the administrator@iprotech.com account.

  • System Events command Server URL: The same as your complete ADD Web URL, for example, https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

  • Platform Web API URL: The same as your complete ADD Web URL, for example, https://addweb.company.com:1125. You are not required to include the port if the port is 443 or 80.

Install Enterprise 2022.10.0 Processing (eCapture) Components

The following sections contain the steps to install the Processing () components:

Processing components should typically be installed in the following order:

  1. Queue Manager to the eCapture Components Machine

  2. eCapture Controller to the eCapture Components Machine

    • Note: You can install the full eCapture Controller to other component machines, but you must install it to the Queue Manager machine as well.

  3. Enterprise Services to the eCapture Components Machine

  4. Limited Controller to a standalone machine

  5. Relativity Import Service to the eCapture Components Machine

  6. File Mounting Service to the eCapture Components Machine

  7. QC/Worker to their own standalone machines

    • Note: QC and Workers should not be installed to the same machines

    • We generally put QC on the eCapture Controller machine as well.

Install Installer Pro

  1. Install InstallerPro on a machine that can access myIPRO (myipro.iprotech.com), or if it’s already installed, launch InstallerPro.

  2. Use your myIPRO account to log in to InstallerPro. You should see the following:

  3. Select the IPRO Add package you want to install and click Next.

    • If you have to make a package installer, select the check box at the bottom left.

  4. For each component, fill out the configuration information as requested (see below for details).

  5. Once the criteria are validated, the component will install or generate the package. To execute a packaged installer, run its PowerShell file on the appropriate machine.

    • Installation packages automatically get saved to
      C:\Users\[USER]\AppData\Roaming\Installer Pro by default.

Queue Manager

DATABASE Fields

  1. SQL Server: The full SQL Instance name where the database is.

  2. Database Name: The name of the database.

  3. SQL Username: The username of the SQL account made for the environment.

  4. SQL Password: The password for the SQL account made for the environment.

  3. Integration Path: The Integration Directory. Must be a UNC path.

  4. System Events Commands Service URL: This is your complete ADD Web URL. (Example: https://addweb.company.com:1125) You do not need to include the port if the port is 443 or 80

  5. Ipro Identity Server Admin Username: The administrator@iprotech.com account

  6. Ipro Identity Server Admin Password: The password for administrator@iprotech.com account. Default is Ipro123!
