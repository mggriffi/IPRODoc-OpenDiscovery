Configure Single Sign-on (SSO)

Administrators can configure to use an external identity provider for Single Sign-on (SSO). If SSO has been configured, users are able to sign in to through the third-party identity provider, rather than having to use -specific login credentials. This removes some of the burdens for users who have to remember multiple login credentials, as well as for administrators who have to manage user security across applications. supports any 3rd party identity provider that uses OpenID Connect. Popular providers include:

  • Google
  • Microsoft
  • Okta
  • Ping

Add a New Identity Provider

Administrators can configure SSO integration in the System Manager. Follow the instructions below to set up the SSO integration:

Note: You can only set up one external identity provider per environment.
  1. Click the Settings icon in the top-right corner of the screen. The Settings icon is a global button that displays in every module of the platform.

  2. The System Manager opens. In the left pane of the System Manager, click Security Settings.

  1. Select the Add New Provider button to the right of Single Sign On.

  2. In the window that displays, configure the fields and select the needed options to set up the integration with the identity provider. For more information on the fields and options available, click here.

    Parameter Name

    Description

    Provider Name

    The name of the external provider (for display purposes only). This name appears on the Environment Settings page, once the provider has been configured.

    Enable Provider

    This option enables SSO integration with the provider. If turned off, users are not able to sign in to by means of this external identity provider.

    Default Identity Provider

    The Default Identity Provider option changes the login screen to automatically redirect to the external provider, preventing users from logging in using a local Identity account.

    Authority URI

    The URI to the external provider

    OpenId Connect Flow

    The flow type (Authorization Code or Implicit)

    Client ID

    The Client ID provided by the external provider

    Client Secret

    The Client Secret provided by the external provider

    Use PKCE

    A Boolean flag to specify whether to use the Proof Key for Code Exchange standard when the OpenId Connect Flow is “Authorization Code”

    Login Screen Button Text

    The button text to appear on the login page. For example, if "Sign in with Microsoft" is used here, the login screen displays as such:

    Subject Claim Name

    The name of the claim in the external provider’s access token that contains the user’s login name or email address

    Authorization Endpoint

    The path for the external provider’s authorization endpoint

    Login Redirect URI

    The URI the external provider will use to redirect the authentication responses to Identity Server

    Logout Redirect URI

    The URI the external provider will use to redirect logout requests

    Token Endpoint

    The path for the external provider’s token endpoint

    User Info Endpoint

    The path for the external provider’s user info endpoint

    End Session Endpoint

    The path for the external provider’s end session endpoint
  3. When you've finished defining the information for the new provider, select Save. The identity provider's information appears beneath the Single Sign On option on the Environment Settings Page.

  4. Having configured your identity provider, you now have the ability to enable your users to log in with the new provider. There are two methods you can use to accomplish this:

    • To enable only certain users, see Enable Select Users for SSO.
    • To enable all users, see Enable All Users for SSO.

Enable Select Users for SSO

If you would like only certain users to sign in with the external identity provider, you can enable them manually in the System Manager. Be advised that users enabled for SSO can no longer sign in with their credentials. To enable individual users, follow the steps below:

  1. Ensure that an external identity provider has been configured in . See Add a New Identity Provider for more information on this step.

  2. Navigate to the Users tab in the System Manager. If the System Manager is not already open, you can open it by selecting the Settings icon in the top-right corner of the screen. In the left pane of the System Manager, ensure the Users tab is selected.

  3. From the list of users, locate the one you would like to enable for SSO. Click on their name. The user profile opens on the right side of the screen.

  4. Click the Edit icon . The user profile can now be modified.

  5. Select the Use External Identity Provider slider. An orange slider means the option has been set.
  6. Click Save in the top-right corner of the screen. Repeat these steps for any additional users that need to be enabled for SSO.

Enable All Users for SSO

If you would like all users in your system to sign in with the external identity provider, you can bulk convert all users simultaneously. Be advised that users enabled for SSO can no longer sign in with their credentials. Review the instructions below for steps on how to bulk convert your users:

  1. Ensure that an external identity provider has been configured in . See Add a New Identity Provider for more information on this step.

  2. Navigate to the Security Settings tab in the System Manager. From this page, select the Migrate Users button to the right of Single Sign On.

  3. The Migrate Users dialog displays. To bulk convert all users to the new provider, select the External Identity Provider option.

  4. Click Convert. A pop-up window appears listing the number of users successfully converted, as well as the number who failed to convert. The names of any users who failed to convert, as well as the reason for why they could not be converted, are also listed in this window.

    Note: The main reason a user would fail to convert is if the account's username isn't an email address; in order to map users from an external provider, the username must match the username (email address) associated with the external provider. This error can be resolved by editing the user account so that the username is the associated email address. After this is resolved, the specifically affected account can be individually made an external account. For instructions on how to edit a user, see Modify User Details.

Modify the Configuration of an Identity Provider

You can change the configuration of an external identity provider at any time. For instructions on how to accomplish this, follow the steps below:

  1. Click the Settings icon in the top-right corner of the screeen. The Settings icon is a global button that displays in every module of the platform.

  2. The System Manager opens. In the left pane of the System Manager, click Security Settings.

  1. Select the icon to the right of the identity provider's name.

  2. The configuration for the provider expands below the name. To edit the configuration, select Edit Provider at the bottom of the screen.

  3. In the window that displays, modify any fields as needed to properly configure the integration with the identity provider. For more information on the fields and options available, click here.

Delete an Identity Provider

If you would like to delete an identity provider, see the following instructions:

  1. Click the Settings icon in the top-right corner of the screeen. The Settings icon is a global button that displays in every module of the platform.

  2. The System Manager opens. In the left pane of the System Manager, click Security Settings.

  1. Select the icon to the right of the identity provider's name.

  2. The configuration for the provider expands below the name. To remove the provider from the system, select Delete Provider at the bottom of the screen.
  3. A warning message displays, indicating that the provider will be permanently deleted. To proceed with the removal, select Yes.
