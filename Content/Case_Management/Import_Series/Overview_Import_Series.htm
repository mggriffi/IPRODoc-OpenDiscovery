Overview: Import Series

An Import Series is created in a Processing Case (a Review case also needs to be created) and contains the fields required for the metadata and the options for the BEGDOC number to be imported into a desired Review Case. The metadata and options include sequential image key numbers, volumes, and like export options (native files, fields, export formats, and so on). These settings are accessed from the processing Case Settings under the Load to Review tab.

For a quick overview of the Load to Review tab, see the below video.

Default Import Series

When you create a Processing and Review case, a default import series is created automatically. If you choose to use the default import series, you do not have to do anything special to be able to start ingesting data. The documents (aka a data set) are automatically added to this import series, and the numbering sequence continues where it left off. For example, if the import series numbering ends with 003000, the new added data set begins its numbering with 003001.

Customized Import Series

You can create a custom import series with its own unique parameters for ingesting data into your Review case. You must mark the customized import series as “active” for data to be ingested with that import series.

For a customized series, you can edit the BECDOC numbers to contain a prefix that might identify a specific data set in the case. As an example, let's say you create a specific prefix "CS" representing the plaintiff “Charles Smith.” The initial numbering sequence would look like CS_000001, CS_000002, and so on. As with the default import series, any new data set that gets added would continue the numbering sequence where it left off.

Note: As you create multiple import series, you can “activate” them in turn based on what data sets you need to import for the Review case. You can only have one active import series at a time.
