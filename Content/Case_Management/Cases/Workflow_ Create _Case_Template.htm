Workflow: Create a Case Definition Template

After you create a case using the default template, the definition of the case can be customized and exported to make it easy to create future cases that share similar characteristics. The load file you create is an XML file with a file extension of .CSE. This file can be selected instead of the default template when creating your case in Case Management. The process of creating the .CSE file is executed within the Eclipse Admin module.

The Case Definition file contains the following case characteristics, but no case data. (A case definition file can be created whether or not data has been added to the case.) All characteristics can be imported into a new case; characteristics marked with an asterisk (*) can be imported into an existing case.

  • Database field definitions (field name and type)

  • Field-level security details

  • Redaction palette*

  • Tags (tag groups and tags)*

  • Relationship definitions*

  • Coding Forms*

  • Searches

Step 1: Review Case Settings

Open the case that you created from the default template and review all case settings to determine what definitions exist in the case. See Manage Review Case Settings for more information.

Step 2: Define Template Options

  1. Select the options you would like to include in your Case Definition File template. The following options can be defined in this step:

    1. Tag palette (tag groups and tags). For more information, see Create Tag Palettes .

    2. Redaction palette. For more information, see Manage Redaction Categories.

    3. Relationship Definitions. For more information, see Overview: Relationships

    4. Coding Forms. For more information, see Overview: Coding Forms.

    5. Saved Searches. For more information, see Work with Saved Searches.

Step 3. Open Eclipse Admin

After you have defined your template options in the review case, the remainder of the tasks are performed in the Eclipse Admin module.

  1. Start Eclipse Admin and log in.

  2. On the Case Management ribbon, select Case Management.

  3. Select your Managing Client and Case.

Define Database Fields

You can add or modify the database fields in your case prior to creating your template for use in a new case.

To change field definitions:

  1. Click the Database Fields tab in the Case Management screen. Field details for the case are listed.

  2. Click  corresponding to the field to be edited. (You may need to scroll horizontally to find this button.)

  3. Make needed changes in the Edit Field dialog box and click Save.

    Note:System fields are not included in .CSE files. If a .CSE file is used to create a new case, new system fields will be created automatically.

Complete the following steps to add new fields:

  1. Click the Database Fields tab. Field details for the case are listed.

  2. Click New Field. The New Field dialog box will open.

  3. Complete the Field Properties for the field.

  4. When finished, click Save.

Define Field Security

By default, all fields are set to Update for a case and no settings appear for groups or users. If no settings are explicitly made for a group, security will be inherited from the case. If field-level security is defined for a group, then these settings will apply to all users in the group. These settings are included in the case definition file.

To change field-level security:

  1. Click the Database Fields tab.

  2. In the Field Security field, select either [Case] or a group name, then click .

  3. Ensure the needed item is selected in case/group list, then select the needed security level for one or more fields (View, Update, or Deny).

  4. When finished, click OK.

Step 4. Export a Case Definition file

  1. Open the Eclipse Admin module. See Step 3. Open Eclipse Admin.

  2. On the Case Details tab, select .

  3. Enter a file name for your new template and save it to a location that is accessible when you create cases in review.

  4. You will get a message that it has been saved. Click OK.

    Note: You can also right-click the case whose definition is to be exported and select Save case definition to a file.
