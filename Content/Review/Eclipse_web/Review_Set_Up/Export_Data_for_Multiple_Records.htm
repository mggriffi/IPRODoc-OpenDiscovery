Export Data for Multiple Records

By default, up to 1 GB of data can be exported from an Eclipse case into a .CSV or .DAT file. Your administrator may configure a different limit. To export data:

  1. Getting started:

    1. Log in to Eclipse and open the needed case (or batch).

    2. In the Case View pane, click Documents.

    3. Optional: Display a specific subset of records by performing a search, opening a Smart Folder, filtering on one or more columns, or selecting an Analytics cluster or category.

    4. Batches only: If you are working in a batch, select the scope of listed documents in the Documents from list.

    5. Hide columns containing data you do not want to export. See "Show/hide and organize column" for details.

  2. On the Documents or related tab’s toolbar, click actions to show the “mass actions” toolbar.

  3. Select the records to be exported:

    1. To export data for all records in the current set (entire case, batch, search results, etc.), select All from the menu in the mass actions toolbar and skip to step 4.

    2. To export data for some, but not all, records, select needed records in the case table as shown in the figure above.

    3. Select Checked from the menu on the mass actions toolbar. In the following example, 25 documents are being selected.

  4. From the adjacent menu, select Export, then click Submit.

  5. In the Export to File dialog box:

    • Output type: select CSV or DAT File.

    • CSV only: select an encoding type: Unicode, ASCII, or UTF8.

  6. Click OK.

  7. In the Save As dialog box, enter a name for the file, navigate to the location where the it should be saved, and click Save.

Note: The file size will be monitored by Eclipse. If it reaches the size limit (1 GB or as configured by your administrator), a message will alert you to the fact and the process will be canceled. To address this issue, reduce the size (select fewer records and/or fewer columns of data), or contact your Eclipse administrator for assistance.
