Working with Private Tags

About Private Tags

Private tags (also called “personal” tags) exist for your personal use. No one else can see the private tags that you create. Use private tags in any way that will help you organize, monitor, and complete your work. For example, you might want to create a set of tags for follow-up work, as shown in this figure.

Private tags are listed in the Private Tags group, which is a “virtual” group that appears on the Document Tags and Page Tags tabs only if you add private tags as explained in this section. Apply private tags as you do public tags (see "Working with Public Tags").

When you are ready to address the documents tagged with your private tags, you can search for them as explained in "Searching for Tagged Documents."

No special permission is needed to create and use private tags. Anyone with access to a case or review batch can create and manage their own private tags.

Note: If you are working on cases from releases prior to Eclipse 2013 that included private folders, documents that were in private folders are tagged with private tags named PF_tagname, where tagname is the name of the former private folder.

Creating Private Tags

To create private tags:

  1. Log In to Eclipse and open the needed case.

  2. In the Case View pane, select Documents, a batch, a folder, or Analytics. Optionally, perform a search for needed documents.

  3. In the Record View pane, select the needed coding form to access the needed tag display.

  4. Right-click a tag group in the Document Tags or Page Tags tab and select Tag Display > Add Private Tag.

  5. Enter a name in the resulting dialog box and click OK. Tags will appear in the Private Tags group as shown in the previous figure. Names must be unique and are not case sensitive. (You cannot have separate tags named “To Check” and “to check.”)

  6. Repeat these steps to add other private tags.

  7. Apply private tags; see "Simple Document Tagging."
