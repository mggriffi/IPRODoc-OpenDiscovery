Tip:

When applying tags, note the following:

For document-level tags, any page that is part of the selected document in the Documents tab can be selected.

Tags applied to a document are not applied to individual pages.

By default, the right-most slideout pane is the Tags pane. It includes two tabs, Document Tags and Page Tags (if the case was defined to allow both types of tags). The type of tag you apply depends on which tab is active, so make sure you are working from the correct tab.

If coding form rules exist, make sure that you follow them as instructed by your administrator.

Simple Document Tagging

To tag individual documents or search results:

  1. Complete "General Tagging Preparation." You must enable tagging as described in that procedure before you can apply tags.

  2. In the tab selected during preparation (Documents, Batch, etc.), click a record for the document to be tagged (or have a tag removed).

  3. Evaluate details for the record to determine which tags to apply (or remove) and complete step 4 or step 5 as needed.

  4. Tag a single (selected) document: In the Document Tags tab, click an unapplied tag to select it (or click an applied tag to remove it). This type of tagging applies the tag to (or removes it from) the selected document according to the rules defined for the tag group.

  5. Tag search results: To tag search results (or Smart Folder, cluster, or similar set of documents that is treated as a search): if one of these items is open in the case pane area and a tag should be applied to (or removed from) all of the documents, right-click the tag and select Tag >Apply to Search Results (or Remove from Search Results).

  6. If a coding form rule is in effect, a message may appear if you select certain tags. Comply with the rule before moving to another document. For example, a rule may require you to add a comment in a Comment field if you select a certain tag.

  7. When tag actions are complete for the selected record, use the Record View toolbar (top of pane), the command bar (bottom of pane), or keyboard shortcuts as described in the following table.

Simple Page Tagging

Note: Most cases use document tagging, but in some instances, page tags may be appropriate. Coding form rules do not apply to page tagging. Check with your administrator if you have any questions about the use of page tags.

Complete the following steps to tag an individual page. This procedure applies when the case is defined to allow page tags and you are in a group that has permission to apply page tags.

  1. Complete "General Tagging Preparation." You must enable tagging as described in that procedure before you can apply tags.

  2. Open the Document Details pane and click the Image tab. If needed, click View on the tab to download images (which makes pages available).

  3. In the tab selected during preparation (Documents, Batch, etc.), click a record for the document whose pages are to be tagged (or have a tag removed).

  4. In the Image tab, select the needed page.

    Note: Single-page documents can be tagged differently at the document and page level. Make sure you apply tags as intended, particularly for single-page documents.

  5. Review details for the page to determine which tags to apply (or remove).

  6. Click the Page Tags tab as shown in the following figure.

  7. Select or clear the tag to be added/removed.

    Note: If nested tags exist, review the behavior explained in "Nested tags".

  9. If you saved changes, navigate to other records using the Record View pane navigation bar as shown in the following figure.

Viewing Applied Tags

You may have the permission to view, but not apply, tags. To view applied tags:

  1. Log In to Eclipse and open the needed case.

  2. In the Case View pane, select Documents, a batch, a folder, or Analytics. Optionally, perform a search for needed documents.

  3. Select a record of interest.

  4. If the Tags pane is not visible, make it visible.

  5. If tags do not appear in the Document Tags tab, complete the following steps. (A specific permission is required.)

    1. Right-click the Private Tags heading and select Tag Display > Modify Tag Display.

    2. Select tags to display and click Apply.

  6. In the Document Tags and/or Page Tags tabs, review tags that have been applied to the selected record.

    Tip: To find documents or pages tagged in a certain way, see "Searching for Tagged Documents."

Creating Public Tags

If you have sufficient permissions, you will be able to create public tags and tag groups as explained in the following paragraphs.

These changes will be available to everyone working on the case or review batch, so it is imperative that changes are made only according to specific instructions from your administrator.

Define New Public Tag Groups

All tags must be in a tag group; groups must exist before you create tags. If you have the permission to do so, create public tag groups in Eclipse as follows:

  1. Log In to Eclipse and open the needed case.

  2. In the Case View pane, select Documents, a batch, a folder, or Analytics. Optionally, perform a search for needed documents.

  3. In the Record View pane, select the needed coding form to access the needed tag display.

  4. Right-click a tag group name in the Document Tags or Page Tags tab and select Tag Display>Add Public Tag/Group.

  5. In the Add Tags and Groups dialog box, click the Add Group.

  6. Enter a name for the tag group. Names must be unique.

  7. Select needed options for the tag group:

  8. Click OK.

  9. Repeat this procedure to add other tag groups.

  10. When finished, add tags to the group(s) as explained in the next procedure. You must add at least one tag to complete the process.

Define New Public Tags

If you have the permission to do so, create public tags in Eclipse as follows. The group to which the tag belongs must already exist. If it does not, complete "Define new public tag groups" first.

Note: Nested (child) tags can also be added to tags, with certain restrictions:

  • If a tag has no child tags and has not yet been applied to documents, then child tags can be added.

  • If a tag has no child tags but has been applied to documents, then child tags cannot be added.

  • If a tag has child tags, additional child tags can be added, whether or not any of the tags have been applied.

  1. Log In to Eclipse and open the needed case.

  2. In the Case View pane, select Documents, a batch, a folder, or Analytics. Optionally, perform a search for needed documents.

  3. In the Record View pane, select the needed coding form to access the needed tag display.

  4. Right-click a tag group name in the Document Tags or Page Tags tab and select Tag Display > Add Public Tag/Group.

  5. Select the needed group name, then click Add Tag. Or, to add tags that are subordinate to an existing tag, select the tag, then click Add Tag.

  6. In the resulting dialog box, enter one or more tag names, one per line. Names must be unique.

  7. When all names are added, click OK.

  8. Repeat this procedure to add tags to other tag groups.

  9. When finished, click Save in the Add Tags and Groups dialog box. Note: By default, new tags will be shown on the default display and added to the affiliated coding form, but not included in the Privilege Log Report (a report available in Eclipse Administration). If changes are needed for these options, do so in Eclipse Administration.

  10. As needed, communicate with your team members to explain the new tags and their use. Administrators should update case instructions and/or alerts as explained in the Eclipse Administration section on "Manage Case Instructions or Documents" and/or "Manage Case Alerts."

Managing Public Tags

Refresh Tag Display

Your administrator or other users may create new public tags (or modify existing tags). Although changes will not appear on the Document and Page Tags tabs (tag display) immediately, shortly after changes are made, you will be notified that the tag display has been updated. If you receive a notification, click the Dismiss Alert button at the bottom of the Document or Page Tags tab and review the changes.

Alternatively, refresh the tag display any time by right-clicking in the Document or Page Tags tab and selecting Tag Display > Refresh Tag Display.

Show or Hide Tags

The following procedure shows or hides the tags visible on the current (Document Tags or Page Tags) tab for the current user. This may be useful in managing your work.

The tags themselves are unchanged and can be added back at a later time.

To add or remove tags from the tag display:

  1. Log In to Eclipse and open the needed case.

  2. In the Case View pane, select Documents, a batch, a folder, or Analytics. Optionally, perform a search for needed documents.

  3. In the Record View pane, select the needed coding form to access the needed tag display.

  4. In the Document Tags tab, right-click any tag name and select Tag Display > Modify Tag Display.

  5. In the Modify Tag Display dialog box, select or clear the check boxes for the tags to be shown/removed from the Document Tags and Page Tags tabs.

  6. When finished, click Apply. The tag display is modified. If you clear all tags for a particular group, the group will also be removed from the display.

Modify or Delete Tag Groups or Tags

The following tag management activities can be accomplished in Eclipse Administration:

  • Renaming tag groups/tags

  • Deleting tag groups/tags

  • Changing tag group rules

For details on managing tag groups and tags, see the page in Eclipse Administration titled "Modify Public Tag Groups or Tags."
