Manage Batch Tags

Applying tags primarily occurs not in cases, but in batches assigned to users who are charged with reviewing documents. To manage the batches that have been assigned to you, you will need to complete the following steps.

  1. Log in to Eclipse and open an assigned batch. (See "Select and Open a Case" for details on opening a batch.)

  2. Complete all review activities as instructed, including tagging for each document.

  3. When an individual document is completed, set the review status as explained in the following procedure, unless your administrator has set a coding form rule that automatically addresses batch status. For example, a rule may be defined to set the status to Reviewed once you complete required tagging activities.

  4. If case instructions call for it, when a review status has been set for all documents in a batch, close that batch as explained in "Checking In a Batch."

  5. Repeat this procedure for all batches.

Setting Review Status

The default status for a document within a review batch is Not Reviewed. A document can also be tagged as Reviewed or On Hold, so that work can be monitored and managed.

Batch review status criteria is determined by your organization (for example, when tagging or redactions are completed). Refer to your case instructions or contact your administrator for information on when to change the review status of a document.

To change the status of a document within a batch (not a case):

  1. Log In to Eclipse and open the needed batch.

  2. Complete needed actions according to your case or batch instructions.

  3. With the Document Tags tab active (not Page Tags tab), select a record in the Documents tab.

  4. Select the status in the Tags pane as shown in this figure.

    Note: The Reviewed option will not be available unless all required tags have been applied and coding form rules (if they exist) followed.

  5. Repeat this procedure to set the status of other documents.

  6. If appropriate, when all documents have been tagged Reviewed and/or On Hold, check in the batch as explained in the next procedure.

Checking In a Batch

To check in batches:

  1. Log In to Eclipse and open the needed batch.

  2. Check case instructions or contact your administrator for details on if and when to check in batches.

  3. Ensure that all records have been reviewed and tagged Reviewed and/or On Hold.

  4. Click Batches in the left pane.

  5. For the batch you have completed (typically indicated by a checkmark in the Active column), click Check In. If the batch has been completed, the status will change to Closed. If not, the status will change to (or remain) In Process.
