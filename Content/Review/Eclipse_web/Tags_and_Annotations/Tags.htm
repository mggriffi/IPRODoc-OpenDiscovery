Managing Tags

About Tags

One of the primary activities when preparing documents for discovery is applying tags. A tag is a type of “marker” that allows you to categorize and identify specific document characteristics.

Typically defined by your administrator, tags allow you and your organization to find and identify similar documents—for example, those that are privileged in some way, or those needing review by a topic expert. Eclipse makes it easy for you to tag documents and/or individual pages. This example shows that a document contains federal tax information.

Tags appear in two tabs in the Tags pane, Document Tags and Page Tags. Your administrator defines each case to include document and/or page tags. If a case is defined to allow only document or page tags, you will see only the applicable tab. If no such limit exists, both tabs will be available.

Note: If Technology-Assisted Review (TAR) projects are implemented at your site, TAR-specific tags will be created. Use these tags as directed by your administrator. For an overview, see the section on "TAR."

Common Uses

Tags allow you to work with documents that share similar characteristics or need similar attention. Following are a few common uses of tags:

  • In Eclipse:

    • Create a Smart Folder for documents sharing one or more tags for further attention – see "Configure Smart Folders."

    • Search for documents by tag – see the following procedure.

  • In Eclipse Administration:

    • Define a review pass based on a specific tag. See "Create Review Passes and Batches."

    • Produce a document set based on a tag. See "Create Export Sets."
