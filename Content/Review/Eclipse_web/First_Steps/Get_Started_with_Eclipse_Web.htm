Eclipse is the review and production component of the platform. Users are able to review, analyze, and annotate case documents with efficient, easy-to-use tools on a Web-based application.

Based on the permissions set by the case administrator, Eclipse users can:

  • View a variety of image, native, and OCR files plus record content, concurrently from a single interface.

  • Add document and/or page-level tags (as set up by the Eclipse administrator).

  • Use advanced and/or visual search to efficiently find case data.

  • Add redactions, markups, and highlights.

  • Edit document text and/or data field content.

  • Use Smart Folders to manage case details or find and organize specific documents.

  • View and annotate case transcripts.

    This section is intended to explain functions typically associated with those who will use Eclipse to access, review, analyze, and annotate case documents in preparation for discovery or other document release purposes. Typical users include case reviewers and experts such as legal counsel. For more information on case set-up and other tasks typically done by a Case Administrator, see IPRO Eclipse Administration.

Before you Begin

Before working in Eclipse, the product installation must be completed and your Eclipse system or case administrator must define you as an Eclipse user. In addition:

  • Your administrator will assign you to a group that is assigned to one or more cases and/or review batches.

  • The abilities you will have are based on the permissions defined for your group in conjunction with the case you access. For example, specific permissions exist for the following:

    • Access to the list of batches on the Batching tab

    • Access details on the tabs in the Document Details pane (image, extracted text, etc.)

  • Tagging, redacting, and other review actions

    • If you have any questions about your permissions, contact your Eclipse administrator.

  • Your administrator must provide you with the Eclipse URL at your company, as well as a user name and password.

Tip: Both name and password are case sensitive—make sure you obtain the correct capitalization. User names and passwords are assigned by the Eclipse system administrator(s). Keep your credentials in a safe place. If you forget your log-in credentials, contact your Eclipse administrator.

Browser Requirements

In addition to having an approved version of Internet Explorer (Microsoft Internet Explorer® 9, 10, or 11), before you access Eclipse the first time, add the Eclipse site to Internet Explorer’s list of trusted sites, and make sure that .NET framework is enabled.

Include in Trusted Sites

  1. Obtain the Eclipse URL from your administrator.

  2. In Internet Explorer, select Tool > Internet Options.

  3. Click the Security tab, then Trusted Sites.

  4. Click Sites as shown in the figure below.

  5. In the Trusted sites dialogue box:

    1. Enter the Eclipse Website's URL.

    2. Optional: clear the verification option (if the protocol is not HTTPS).

    3. Click Add, as shown in the figure below.

  6. Close the dialogue box.

Enable .NET

  1. In Internet Explorer, select Tools > Internet Options.

  2. Click the Security tab and select Trusted sites.

  5. Click OK to close the dialogue box.

Administrative Access

Users belonging to the Super Admin group can open all cases in Eclipse. They can also open (and will be assigned) unassigned batches in any case, or open other users’ batches.

Users who are not in the SuperAdmin group but have been given case management permissions can open all cases to which they have been assigned. In addition, they can open (and will be assigned) unassigned batches in their cases, or open other user’s batches.

Working Sessions

Once you start Eclipse, your working session will remain active as long as you work in the application.

If there is a pause in your activity, the session may time out; a message will alert you that the session is idle. If this happens, click OK in response to the message and log in again.
