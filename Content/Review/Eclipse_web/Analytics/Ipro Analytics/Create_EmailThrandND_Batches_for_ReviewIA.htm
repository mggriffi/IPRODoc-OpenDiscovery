Create Email Thread with Near Duplicate Batches for Review ()

You can combine Email Threading and Near Duplicate technology to create Batches for Review. This allows you to use Email Threads to review emails and their attachments, and Near Duplicate technology to review the remaining loose documents.

  1. Follow the process for setup of Full Thread or Inclusive Email Batch Review. For more information, see Create Full Email Thread Batches for Review (IPRO Analytics) or Create Inclusive Email Thread Batches for Review (IPRO Analytics).

    		2.

    Note: Email Thread batches must be created prior to creating the Near Duplicate batches for the criteria to work properly.

  2. Two Advanced Searches will be required for creating Near Duplicate batches.

    1. First Search: Create a new Advanced Search entitled Email Thread Batches.

      The criteria of the search should identify all documents that are part of the Email Thread review pass.

      This can be accomplished by searching for any documents where the (Batch Set) is not the Email Threading review pass previously created.

    2. Second Search: Create a new Advanced Search entitled Near Duplicate Batches.

      The criteria of the second search should identify all documents that received a Near Duplicate value, that are not part of the Email Thread Batches search created above.  

      This can be accomplished by searching for any documents where the IA ND Family field contains a value AND the (Saved Searches) Is NOT In [Email Thread Batches].

  3. Create a new Review Pass and identify your Advanced Search for the documents to be batched.

    1. Enter a Batch Prefix

    2. Enter the Batch Size of your choice

    3. Set the Family Field to IA ND Family to ensure all Near Duplicates are kept together

    4. Choose the Saved Search entitled Near Duplicate Batches

    5. Sort your Review Pass by IA ND Sort ascending

    6. Select Save

    7. Select Create Batches in the bottom right corner

      Important: Using these techniques together will still not cover the entire review set.  Documents that are not part of email threads and are not part of near duplicate groups will still need to be batched.
