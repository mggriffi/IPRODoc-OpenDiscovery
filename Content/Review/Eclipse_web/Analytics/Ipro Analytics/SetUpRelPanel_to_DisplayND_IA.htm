Set Up the Relationship Panel to Display Near-Duplicates ()

The Eclipse near-duplicates component, based on Analytics textual processing, identifies textual near-duplicate groups in item text. Textual near-duplicates are items in which most of the text appears in other items in the group and in the same order.

The metadata of the items identified as part of a near-duplicate group will be updated with a group ID to identify the near-duplicate group to which the items belong.

Note: Near-duplicate detection will typically be configured by you case administrator. For more information, contact your case administrator or see Configure Near-Duplicate Detection in IPRO Analytics in the Eclipse Administration section.

Set Up the Relationship Panel to Display Near-Duplicates

In order to see near-duplicates during your review process, you must first set up the Relationship panel, in Eclipse Web, to display near duplicates. .To do so:

  1. In Eclipse Web, navigate to Case Administration and then Relationships.
  2. Review the list for the Near Duplicates panel. If it does not exist, create one by clicking on the Plus (+) button and entering Near Duplicates in the name field. If the panel does exist, ensure the panel is set up as described below.
  3. Under Group By, select the IA ND Family field or the equivalent field you created.
  4. Under Sort By, select IA ND Sort field, or the equivalent field you created.

  5. Be sure to check the box to Show document compare tool so that comparing the text in the Near Duplicate documents is easier.

  6. In the Selected Fields section, add the following fields, at a minimum, to be displayed in your view and select Save.

After you have configured near duplicate detection in Eclipse Administration and you have set up the relationship panel to view near duplicates, you can:

  • Set up batch review by near duplicate group to allow for documents in the same ND group to be reviewed by one reviewer all at the same time, providing for faster, more consistent review decisions. For more information, see Create Near Duplicate Batches for Review ([%=Primary.Analytics%]).
  • Use Email Threads to review emails and their attachments, and Near Duplicate technology to review the remaining loose documents. For more information, see Create Email Thread with Near Duplicate Batches for Review ([%=Primary.Analytics%])
