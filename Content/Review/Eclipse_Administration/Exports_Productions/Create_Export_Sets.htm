Create Export Sets

The procedures explained in this topic are used after you have completed preparation. (See Understand Export and Production and the section Before You Begin)—whether you are:

  • Exporting Eclipse case data/files,

  • Loading data directly into Relativity, or

  • Creating a production set

Getting Started

Begin defining an export set with the following steps:

  1. Start Eclipse and log in as an Eclipse administrator.

  2. Open the needed case.

  3. In the Case View pane, click Case Administration, and then Export Sets.

  4. Click .

  5. Information: Complete basic export-set information shown in the table on the following page.

  6. Continue with the procedures explained in this section.

Click  or  to expand or collapse each section. Save an export set definition at any time by clicking Save at the bottom of the form. Continue work on an unfinished definition by selecting it in the list of export sets and clicking .

Basic Export Information

Option

Description

Name

Enter a name for the set.

Date

Enter the date the export (or production) job is estimated to be completed or delivered, or click  and select the date.

Use a date format of MM/DD/YYYY. A single digit for month and/or day is acceptable.

Output Location

Enter the full path for exported files, or click Browse and navigate to/select the needed folder.

IMPORTANT: Make sure the following conditions are met:

The location is in UNC format (such as \\server001\exports\smith-jones\).

The destination folder exists.

Eclipse has access to the location (for example, it is in the same or a trusted domain).

If you will be loading directly into Relativity, the output location is still required. Loading to Relativity also exports a .DAT file, which will be placed in the output location.

Export Environment

Starting with Eclipse 2016.3.0, all export and production jobs are processed using eCapture and Workers.

Click in the Export Environment field and select the eCapture environment for this export/production job. The environments available are based on the application environments defined in .
