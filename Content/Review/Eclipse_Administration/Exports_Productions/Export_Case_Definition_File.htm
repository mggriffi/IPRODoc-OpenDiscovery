Export Case Definition File

After you create a case, the definition of the case can be exported to make it easy to create future cases that share similar characteristics. The load file you create is an XML file with a file extension of .CSE.

It contains the following case characteristics, but no case data. (A case definition file can be created whether or not data has been added to the case.)

All characteristics can be imported into a new case; characteristics marked with an asterisk (*) can be imported into an existing case.

  • Database field definitions (field name and type)

  • Field-level security details

  • Tagging options (document/case level tagging)*

  • Redaction palette*

  • Tag palette (tag groups and tags)*

  • History/tracking options

  • Stop Words

  • Relationship definitions*

  • Coding forms*

Export Procedure

To export a case definition file:

  1. Start Eclipse Administration and log in as a user who is a member of the Super Admin group.

  2. On the Case Management ribbon, select Case Management > Case Management.

  3. In the Case Management screen, select the managing client for the case whose definition is to be exported.

  4. In the Case Name list, right-click the case whose definition is to be exported and select Save case definition to a file.

  5. In the resulting dialog box:

    1. Navigate to the location where you want to save the case definition file.

    2. Enter a file name and click Save. The file extension is not required.

  6. In response to the confirmation message, click OK. When appropriate, use the exported case definition file to create a new case as explained in Create and Manage a Case or to assist in the configuration of an existing case as explained in Import Case Objects.
