Complete the Export

To validate and run an export job after the previous procedures are completed:

  1. Get Started:

    1. If a saved public search representing the documents to be produced does not exist, create and save the needed search as explained in Saved Searches.

    2. If Production Shield has not been configured and you want security applied to this export job, complete Configure Production Shield.

    3. If the export set is saved/not open, in the Case View pane, click Case Administration > Export Set. Select the needed set in the list and click .

  2. In the Export Tools section of the screen, click Select Documents.

  3. In the Select Documents dialog box, select the needed saved search.

  4. Click Validate and evaluate the Validation Report. A typical report is shown in the following figure; different entries will exist depending on your export selections.

  5. If the report identifies conflicts, either export the report by clicking Export or close the report and refer to Summary details in the right pane of the export-set form. Possible issues and override details are explained in the following table. In the table, Override indicators are as follows:

    6. Y = Can be overridden

    S = Can sometimes be overridden (depending on circumstances)

    N = Cannot be overridden

    Export/Production Troubleshooting

    Conflict

    Override?

    Description

    Missing natives

    S

    One or more native files in the production set do not exist in the location defined for the case.

    Can be overridden if you are exporting both images and native files, in which case missing native files will simply be skipped.

    Cannot be overridden if you are producing only native files.

    To correct this conflict, take such actions as placing needed native files in the correct location or changing the export set to exclude those documents.

    Missing images

    N

    No image is associated with one or more documents in the export set (no image path is defined).

    Correct this conflict by actions such as adding a custom placeholder for records lacking an image path, or performing a global replace for the image path. For details, see:

    • "Select Output Locations" step 3

    • "Globally Replacing Image Paths

    NOTE: Other image problems are not considered to be missing images. For example, an incorrect file path, an image missing in the specified location, server problems, etc. They are indicated in the production export by an “Error Processing Image” placeholder.

    Make corrections to these types of problems on a case-bycase basis, depending on the type of problem.

    Missing text

    Y

    One or more text files in the export set do not exist in the location specified for the files.

    If this problem is overridden, a “Missing Text” placeholder page will be included for each missing text file.

    Alternatively, correct this conflict by such actions as placing needed text files in the defined location or changing the production set to exclude those documents.

    Shield conflicts

    Y

    Export documents are checked for all defined rules.

    Before overriding this issue, review the rules and documents in the document set to make sure overriding is the correct action.

    Valid output path

    N

    The defined Output Location cannot be verified. Correct the location. Make sure the following conditions are met:

    • The location is in UNC format (such as \\server001\productions\ smith-jones\).

    • The destination folder exists.

    • Eclipse has access to the location (for example, it is in the same or a trusted domain).

    • A volume with same name as the one defined for this production set does not already exist in the output location.

    Valid native naming field

    N

    Occurs if naming native files by a field and the field contains no data (or only spaces or an empty string) for some documents.

    Either change the way native files are named or edit the empty fields so that they contain the needed data.

    Broken family

    Y

    Include Family is part of the saved search used to specify documents for the export set, and not all family documents can be located.

    Before overriding this issue, review documents in the document set to make sure overriding is the correct action.

    Bates Number conflict

    N

    This conflict will occur a) only if Document Numbering is Use image priority and b) only if documents with duplicate numbers have been selected from the various document sets (original image and/or production export jobs chosen for this export).

    Resolve duplicate numbers by such actions as changing the order of document sets selected for numbering or choosing a different type of numbering.

  6. Continue as follows:

    • If conflicts were found, make needed corrections and/or override the conflicts as required. then go to step 7.

    • If no conflicts were found, go to step 8.

  7. After resolving or overriding conflicts, click Validate to re-check the export set.

  8. After all conflicts are resolved and/or overridden, click Export to complete the job. (The Export button is available only when all conflicts have been addressed.)

  9. Observe Export Status below Export Summary in the right pane. Status will change as process steps complete, and may include the terms listed in the following table.

    10. Export Status Details

    Status

    Description

    Not Started

    The export process has not yet begun.

    Staging

    The export data is being placed in the staging database.

    Staging Complete

    All export data is in the staging database.

    Pending

    The export job has been placed in a queue.

    In Progress

    The export data is being processed.

    Paused

    The export job has been placed on hold.

    Deleting

    The export job has been canceled and Eclipse is retrieving data about the job, such as errors and information about the status of the job at the point of cancellation.

    Deleted

    Cancellation has been completed.

    Complete

    The production job has finished; production data may be transferring from the staging database back to Eclipse or the data transfer may have finished.

  10. When the export job is complete, export files are ready for use, either in Eclipse or in Relativity, depending on your export selections.

  11. If errors exist, click the Total Errors number (a link). A Production Errors dialog box opens and lists documents, page, and load file error information. Details can be exported to a .CSV file if needed.

  12. As needed, review production related-reports. See Eclipse Web End User for details on working with reports.
