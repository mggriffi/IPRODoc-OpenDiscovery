Configure Production Shield

Note: These procedures apply for versions prior to 2018.6.0. See Set the Production Shield for version 2018.6.0 and later..

After you have prepared for export/production security, complete the following steps:

  1. Get started:

    1. Start Eclipse and log in as an Eclipse administrator.

    2. Open the needed case.

  2. In the Case View pane, click Case Administration, and then Production Shield.

  3. Click , then enter a name for the security configuration.

  4. To include family documents as part of the configuration, select Include (field). The field named here is the BEGATTACH field for your case. If this option is selected, all documents related to those matching your security criteria will be flagged as protected.

    5. The following steps are very similar to those used to define an advanced search in Eclipse.

  5. From the Search Type list, select the first type of security criteria:

    • Fields within the case

    • Analytics (if installed at your site) - Category or Clusters

    • Batch - Assigned To, ID, Review Status, or Set

    • Doc Tags

    • Redaction Categories

    • Review Status

  6. Complete the security criteria statement by selecting the needed Operator and selecting or entering the needed value below the statement.

  7. Optional: Add another statement(s) by selecting the needed Connector, clicking +, and repeating step 5 and step 6 for the new statement.

  8. Optional: Group two or more statements by using the parentheses buttons.

  9. When finished, review the complete security expression and make changes to individual statements if needed.

    10. To make changes before the configuration is saved, simply change any aspect of a security statement (search type, operator, value, connector), or delete a statement by clicking - for the line.

  10. When the expression is correct, click Save.
