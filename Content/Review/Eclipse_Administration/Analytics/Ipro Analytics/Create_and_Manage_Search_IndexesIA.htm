Create and Manage Search Indexes

This topic explains how to create and manage analytics indexes used in Analytics. To learn more about Content Filters and Indexes, watch the Content Filters and Indexes video.

Click here to watch the Content Filters and Indexes video

Before you begin

Planning will bring you the greatest success with custom search indexes for each case. To familiarize yourself with needed information and available options, “walk through” the index management function as follows:

  1. You need to understand your case data and reviewer tasks. Review cases and discuss with reviewers the types of searches that will assist them as they do their jobs.

  2. Start Eclipse Administration and log in as an Eclipse administrator.

  3. Before you begin, ensure any IPRO Analytics Content Filters have already been created.

  4. On the Case Management ribbon, select IPRO Eclipse Analytics > Manage Analytics Indexes.

  5. Select the needed managing client and case.

  6. Continue with the procedures explained in this section.

Create an Analytics Index

The Create New Analytics Index dialogue allows you to specify various options for building the index. To create a new Analytics Index:

  1. Getting started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select IPRO Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client and case.

  2. Click Create New Analytics Index.

  3. Enter the following basic information in the top portion of the dialog box.

    Item

    description

    Index Name

    Enter a name for the index profile.

    Documents to Index **

    • To index all documents in the case, select <Entire Case>.

    • To index a subset of documents based on a search saved in Eclipse, select the search and click OK.

    Configuration

    Select the configuration profile you would like to use.

    For more information about Configurations, click here to watch the Configurations video

** When choosing a subset of documents to be indexed, keep in mind that the indexing process for Analytics has several criteria that are automatically applied. This helps to ensure that the documents submitted for indexing will provide the most valuable results:

  • Content included in your IPRO Analytics Content filters are automatically excluded from the index.
  • Documents automatically identified as Short or Binary during indexing will be listed as such in the IA Doc Treatment field and are automatically excluded from the index.
  • Extremely large text files (> 4MB) are not automatically excluded, but if there is an abundance, it may result in the index building more slowly. You might want to consider excluding these from your search criteria if they are in question.

Create a New Configuration

If you would like to create a new configuration profile, click New Configuration and enter the appropriate information in the following fields:

Item

description

Name

Enter a name for the configuration profile.

Text Assumption Level

Select one of the following options to determine how documents will be treated when analyzing for non-binary documents (i.e. documents that contain readable text as opposed to just numbers):

  • 0: Only treats documents as non-binary after the system has analyzed the text and identified the content as containing prose.

  • 5: If the document was converted from a known format to plain text, it assumes that the result should be treated as non-binary.

  • 10: In addition to the assumptions made at level 5, if the document is in a format that can’t be translated into text, it assumes that the document can be treated as non-binary without further translation of validation.

Similarity Cutoff

Enter the Similarity Cutoff value (from 1 to 100) to be used for determining the threshold for how similar two documents can be. The default value is 25.

Insignificant Words

Default list of stop words (commonly used words that should be ignored, such as in, or, of). Words may be added or removed from this list.

Significant Words

Add words that you want to ensure the index pays attention to and are known to be important for determining relevance, or words that you might want to include in analytics searching.

Manage Analytics Indexes

Edit or Delete an index

  1. Getting started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select IPRO Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client/case.

  2. Edit an Index:

    To edit an existing index, click corresponding to the necessary index. In the Edit Analytics Index dialog, make any needed changes in the following fields:

    item

    description

    Index Name

    Edit the name of the index profile.

    Documents to Add

    Documents can be added to an index by selecting one of the following options:

    • To add all documents in the case to the index, select <Entire Case>.

    • To add a subset of documents based on a search saved in Eclipse (including private searches), select the search and click OK.

    Note: If there are no new documents to be processed in your selected search, you will be prompted to select one with unprocessed documents before you are allowed to proceed.

  3. Optional: To delete an existing index, click the corresponding to the index you wish to remove from your case.
