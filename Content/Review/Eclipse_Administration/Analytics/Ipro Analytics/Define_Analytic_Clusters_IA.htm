Define IPRO Analytics Clusters

Clustering is a means of grouping conceptually similar documents together in a tree-type hierarchy. Eclipse automatically applies a descriptive title to each cluster of documents without requiring user knowledge or input within the data set.

  • Review Prioritization by Topics - You can use the Concept Wheel or Concept tree to identify and tag high and low priority concepts within the data set. This can then be used when batching documents for review.
  • Batch by Topic Review - You can set up batch review that is organized by conceptual topics to allow for documents containing conceptually similar text to be reviewed by one reviewer all at the same time, providing for faster, more consistent review decisions. For more information on creating review passes and batches, see Create Review Passes and Batches.

Before You Begin

Planning will bring you the greatest success in defining Analytics for each case. To familiarize yourself with needed information and/or available options, “walk through” the Analytics configuration function as follows:

  1. You need to understand your case data, case documents, and reviewer tasks. For example, review cases and discuss with reviewers the types of clusters and categories that will assist them as they do their jobs.

  2. For categories: If an appropriate analytics index has not been created, complete Create and Manage Search Indexes.

    1. Determine needed category sets. Sets are distinguished by the number of allowed documents, similarity score, and/or which search is to be affiliated with the category set. For example, you may have the category set “Energy Issues.”

    2. Determine needed categories, that is, titles that represent the topics users will need to find in Eclipse. Categories should be based on specific topics. For example, you may have categories of “Gas,” “Nuclear,” “Solar,” and so on.

    3. Identify examples of content that exemplify the categories you create. Examples may be case documents, document text, and/or other content.

  3. Start Eclipse Administration and log in as a user who is a member of the Super Admin group, then select each command from the Analytics menu that you will be using. Review the configuration screens and plan your approach.

    Note: Some procedures require that you work in both Eclipse Administration and Eclipse (web).

Define Analytic Clusters

After completing preparation as described in Before You Begin define clusters as follows:

  1. Start Eclipse Administration and log in as a user who is a member of the Super Admin group.

  2. On the Case Management ribbon, select IPRO Eclipse Analytics > Manage IPRO Eclipse Analytics Clusters.

  3. On the left side of the screen, select the needed managing client and case.

  4. Click Create New Cluster and complete the Cluster dialog box as described in the following steps.

  5. Cluster Name: Enter a unique name that describes the conceptual similarity to be reflected in the cluster.

  6. Documents to Cluster: Select either the entire case or a saved search (results of a search saved in Eclipse).

  7. Settings: Complete this section as needed. Default values exist for all items.

    Item

    description

    Cluster Levels

    The number of levels (rings) the documents will be clustered into. Select a cluster depth value from 1 to 7. The default value is 4.

    This value defines the maximum hierarchy depth for clusters when they are created. It is used to determine if subclusters will be generated on any computed cluster.

    No subclustering will be done if the computed cluster is already small, (usually less than 10 items).

    Root Clusters

    If needed, enter or select the number of root clusters from 15 to 30. The default value is 16.

    • Lower settings: generate clusters based on a relatively low level of conceptual similarity.

    • Higher settings: generate clusters based in a relatively high level of conceptual similarity.

    If your case is small or contains few identifiable topics, the number of root clusters may be lower than 15.

    Normalize Docs

    If needed, select if word normalization/lemmatization will be used on your cluster set. This box is selected by default.

    Word normalization should improve the accuracy of clustering by determining the lemma (headword relative to the definition) of a word based on its intended meaning. This helps improve the handling of words which have multiple meanings.

  8. When selections are completed, click OK. Clusters are created and will be available in Eclipse.

  9. Repeat this procedure as needed to create other clusters with different configuration settings.

  10. Alert your users to the availability of the new clusters. Explain how the clusters can help them locate needed information as they review documents.
