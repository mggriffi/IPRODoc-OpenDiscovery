Define IPRO Analytics Categories

Categorization refers to the identification of conceptually relevant documents by using a small subset of documents as examples for each user-defined category. Once examples are identified, the system will analyze all other documents and group them into one or more categories based on conceptual similarity.

Categorization will increase document review efficiency by identifying and grouping documents into relevant user-defined categories.

Before You Begin

Planning will bring you the greatest success in defining Analytics for each case. To familiarize yourself with needed information and/or available options, “walk through” the Analytics configuration function as follows:

  1. You need to understand your case data, case documents, and reviewer tasks. For example, review cases and discuss with reviewers the types of clusters and categories that will assist them as they do their jobs.

  2. For categories: If an appropriate analytics index has not been created, complete Create and Manage Search Indexes.

    1. Determine needed category sets. Sets are distinguished by the number of allowed documents, similarity score, and/or which search is to be affiliated with the category set. For example, you may have the category set “Energy Issues.”

    2. Determine needed categories, that is, titles that represent the topics users will need to find in Eclipse. Categories should be based on specific topics. For example, you may have categories of “Gas,” “Nuclear,” “Solar,” and so on.

    3. Identify examples of content that exemplify the categories you create. Examples may be case documents, document text, and/or other content.

  3. Start Eclipse Administration and log in as a user who is a member of the Super Admin group, then select each command from the Analytics menu that you will be using. Review the configuration screens and plan your approach.

    Note: Some procedures require that you work in both Eclipse Administration and Eclipse (web).

Define Analytic Categories

To define categories:

  1. Complete "Before You Begin

  2. On the Case Management ribbon, select IPRO Eclipse Analytics > Manage IPRO Eclipse Analytics Categories.

  3. Select the needed managing client and case.

  4. Create a Category Set:

    1. Click New Analytic Category Set.

    2. Complete the resulting dialog box with the following details and click OK.

      Item

      description

      Set Name

      Enter a name for the category set.

      Analytics Index

      Selecting the analytics index for which the categories are being created.

      Minimum Similarity

      Enter the similarity score (1 to 100).

      This score expands or narrows the evaluation for matching similar documents. A higher number represents higher precision but less recall; a lower number represents lower precision but higher recall.

      Maximum Documents

      Enter the maximum number of documents to be added to the Category Set.

      Search Index

      Select the analytics search index for which the categories are being created.

  5. Create a Category:

    1. Click New Analytic Category.

    2. Enter a Category name and click OK.

    3. Repeat these steps to create additional categories.

  6. Provide examples for a category—complete step 7 and/or 8 to add examples that exemplify the category.

  7. To add examples in Eclipse Administration:

    1. If example content is in a source document, open the document and copy the needed text.

    2. For the category for which examples are being added, click the corresponding.

    3. In the Analytic Example dialog box, type or paste (Ctrl+V) the example content. NOTE: If you type the example, do so as a single paragraph. Do not press the ENTER key.

    4. Click OK.

  8. To add examples in Eclipse:

    1. Log in to Eclipse and open the needed case.

    2. From the Case View pane, select Documents, a Private or Smart Folder, a specific batch, or Analytics. Or, conduct a search such that search results display in the case pane area.

    3. Select the record for the document that contains the example text.

    4. Open the Extracted Text or Native Viewer tab; if the document is not visible, click View on the tab.

    5. To add the entire document as a category example, right-click in the Extracted Text or Native Viewer tab and select Add Doc to Category.

    6. To add selected text as a category example, select the text in the tab, then right-click and select Add Text to Category.

  9. Repeat these steps to add other category examples; when finished, go to the next step.

  10. Build the categorization index:

    1. Log in to Eclipse (web) and open the needed case.

    2. Click Analytics in the Case View pane.

    3. Expand the Categories list and locate the applicable category group.

    4. Right-click the category set and select Build Category as shown in this figure.

      Note: The progress will be displayed as (Building) to the right of the category set name until it is completed.

  11. After the categories are built, alert your users to the availability of the new or re-built categories. Explain how they can use categories as they review documents.

    • Viewing Category Results - Once the Build is complete, the results can be viewed by clicking on each category name within the category set on the Analytics menu.

      Note: Additional examples can be added at any point in the process by following the steps above and rebuilding the category set.
    • Searching for Category Results - Within Advanced Search, under the Search Type dropdown, select (Analytics Category), choose an operator and select the category set(s) and categories to be searched for.
