Basic TAR Report Procedure

To run a TAR report:

  1. Log in to Eclipse and open the needed case or batch.

  2. In the Case View pane, click Reports.

  3. Click the needed TAR report.

  4. Make additional selections as required by the report.

  5. After the report is generated, review results in the Reports tab. Use the following actions as needed:

    • Scroll vertically and/or horizontally to view all information.

      Click in the report and press the PAGE UP and PAGE DOWN keys for easy scrolling.

    • Click a column heading to sort by that column.

    • Enlarge or reduce a column’s width by dragging the right or left edge of the column’s heading.

  6. To the report, see the following procedure.

Export a TAR Report

To export a TAR report to a .CSV file:

  1. Run the report as explained in the previous procedure.

  2. At the bottom of the Reports tab, ensure that Export to CSV is selected in the actions list.

  3. Click Run.

  4. In the Save As dialog box, navigate an appropriate location for the file, enter a file name, and click Save. It is not necessary to include the file extension (.CSV).

  5. Open the file in a spreadsheet or other appropriate application.

TAR Report Details

Refer to the following tables for details about the data presented in TAR reports. Tables are listed in alphabetical order; data in each table is in the order it appears in each report.

TAR Project Certification Details Report

DAta

Description

Review Round

Certification Round (the type of review pass).

Batch Name

The name of the certification batch.

Review Document Total

Number of documents reviewed in the certification round.

Confirmed Non Responsive

Number of documents tagged as Non Responsive by the (human) reviewer.

% Combined Non Responsive

Percentage of documents in the certification round that were tagged as Non Responsive.

Certification Error %

The margin of error defined for the certification round, expressed as a plus/minus range (for example, +-3).

Estimated Missed Responsive Documents

The estimated number of responsive documents remaining in the TAR Project that have not been categorized as Responsive. This value is based on a formula that considers the following factors:

  • % Combined Non Responsive

  • Certification Error %

  • Number of documents in this TAR project categorized as Non Responsive.

TAR Project Validation Details Report

Data

Description

Review Round

Validation Round (the type of review pass).

Batch Name

The name of the batch.

Responsive - True

The number of documents for which the (human) reviewer and the Eclipse TAR analysis both tag the documents as Responsive.

Non Responsive - True

The number of documents for which the reviewer and the TAR analysis both tag the documents as Non Responsive.

Responsive - False

The number of documents for which the reviewer tags the documents as Responsive, but the TAR analysis considers the documents Non Responsive.

Non Responsive - False

The number of documents for which the reviewer tags the documents as Non Responsive, but the TAR analysis considers the documents Responsive.

Precision[1]

A measure of the accuracy of the identification of document responsiveness.

For validation rounds, this applies to documents tagged as Responsive. A higher precision value means a more accurate result, that is, that the documents that have been reviewed are indeed Responsive.

Recall*

A measure of completeness that is based on the number of documents tagged as Responsive as compared to the total number of truly responsive documents.

F Measure*

A method for factoring both precision and recall to indicate how well the TAR process has identified truly Responsive documents.

Sample Type

The type of sample used, Fixed or Statistical, plus values related to the sample type:

For Fixed, the number of documents selected for the sample will be listed.

For example, Fixed/40.

For Statistical, the selected confidence level and margin of error will be listed.

For example, Statistical/95, +-3

[1]Values are based on standard statistical calculations for Precision, Recall, and F Measure.
