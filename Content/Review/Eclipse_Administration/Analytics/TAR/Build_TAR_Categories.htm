Build TAR Categories (Content Analyst)

The category set created as part of a TAR project is named TAR_ABC_CAT, where ABC is the prefix defined for the TAR project. Categories include:

  • TAR_ABC_Responsive: Documents identified as relevant to the issues/needs of the case by either Eclipse analysis or human review.

  • TAR_ABC_Non Responsive: Documents identified as not relevant to the issues/needs of the case by either Eclipse analysis or human review.

Before each validation round, the TAR category set should be rebuilt.

To rebuild TAR categories:

  1. Get started:

    1. Open the case for which the TAR categories are being created/updated.

    2. Start Eclipse and log in as an Eclipse administrator.

  2. In the Case View pane, click Analytics.

  3. Expand the Categories list and locate the needed TAR category set.

  4. Right-click the category set and select Build Category as shown in this figure. Wait as documents are added to the categories.

  5. As needed, review document details in the resulting Search Results tab or Concept Wheel.

  6. If you will conduct a review of all documents (for example, a prioritization review) instead of a certification round after validation rounds are complete, use the Responsive and Non Responsive categories to create final review passes.

For details on category management, see Define IPRO Analytics Categories.

Notes on building categories

Duplicate documents

Duplicate documents may be evaluated (by human reviewers) as part of the TAR process. TAR categorization impacts all reviewed documents, including duplicates. As part of the TAR process, all duplicate documents are categorized in the same way, based on the last duplicate document reviewed. Therefore, discrepancies between a document's tag and its category may result if reviewers tag duplicate documents differently.

For example, if a reviewer tags document A as Responsive, then tags duplicate document B as Non Responsive, the documents will have different tags. After TAR categories are built, however, both documents will be categorized as Non Responsive (based on the last document reviewed, document B).

Issues such as these are listed in the TAR Project Discrepancies report; discrepancies can be corrected by re-evaluating the discrepant documents, tagging them consistently, then rebuilding TAR categories.

Uncategorized documents

Documents with coherency scores below the selected maximum are not categorized. Documents with very little text, such as multimedia files or spreadsheets containing primarily numeric data, are the types of documents that often have a low coherency score.

To keep uncategorized documents to a minimum, ensure the Analytics index used for the TAR project is optimized to find only the documents containing the needed concepts.

To find and review all uncategorized documents, use Advanced Search to find all documents for which the project’s “TAR Score” field value is less than the category’s Maximum Coherency Score value.
