Train the TAR System (IPRO Analytics)

The field created as part of the TAR project is named TAR_ABC_Prediction, where ABC is the prefix defined for the TAR project. Values for this field include:

  • Predicted_Relevant: Documents identified as relevant to the issues/ needs of the case by either Eclipse analysis or human review.

  • Predicted_NON_RELEVANT: Documents identified as not relevant to the issues/needs of the case by either Eclipse analysis or human review.

  • PREDICTION_UNDETERMINED: Documents that could not be identified as relevant or non-relevant because the document was mostly binary data or contained little or no words.

After a TAR batch has been marked as completed, the system is able to learn from the decisions made by a human reviewer. Before each TAR round, the previous round should be checked in as completed.

To train the TAR System:

  1. Getting started:

    1. Start Eclipse and log in as an Eclipse administrator.

    2. Open the case for which the TAR categories are being created/updated.

  2. In the case view pane, click Batches.

  3. Locate the completed TAR Batch.

  4. If necessary, mark the batch as Active.

  5. Left-click the Check In button and select Completed.

  6. Click OK. Wait as the batch is checked in.

    Note: If you will be conducting a review of all documents (for example, a prioritization review) instead of a certification round after validation rounds are complete, use the Predicted_Relevant and Predicted_Non_Relevant fields to create final review passes.
