Provide Instructions for Reviewers

TAR Coding Forms

As part of TAR preparation, familiarize yourself with the TAR coding form, including its fields, tags, and rules. The name of the coding form is TAR_ABC, where ABC is the prefix defined for the TAR project.

See Manage Coding Forms for details on coding form elements, but remember that it is recommended that you do not modify the TAR coding form, except possibly as noted:

  • If you want users to review documents for issues other than TAR responsiveness, you can add needed tags to the form.

  • If you want other fields to be in the coding form, add those fields.

Note:

  • Do not add TAR-specific fields that might sway a user’s review decisions (for example, do not add the TAR Score field).

  • Do not apply security to TAR coding forms; control access to these forms by defining TAR security as explained in step 8 of TAR Project Configuration.

Instructions for Users

Prior to each review cycle, provide detailed instructions to the reviewer(s) who will evaluate TAR project documents, including such details as:

  • Coding form: name of the coding form.

  • Tagging: criteria for using the TAR tags (Responsive, Non Responsive, and Do Not Use As Example).

    • Note: Users may also apply other (non-TAR) tags when they are performing the TAR review. If this should be done, provide guidelines for this tagging as well.

  • Example excerpts: details about when and what to include in the TAR_ABC_ExampleExcerpt field (where ABC is the TAR project prefix). Typically, when only a specific (narrow) portion of a document is Responsive, that content is copied from the Native Viewer or Extracted Text tab and pasted into the ExampleExcerpt field.

    Note: Example excerpts is available for Content Analyst only and is not available for .

  • TAR categories: criteria for adding document content or an entire document to one of the TAR categories.

Refer users to the following sections of Eclipse End User Help for assistance:

  • Work in the Record View pane (using coding forms)

  • Tag documents (applying tags)

  • Work with case documents (copying text from documents)

  • Use Content Analyst (adding content for categories)
