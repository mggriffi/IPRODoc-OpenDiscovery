Change Case Details

Case details that can be modified in Eclipse Administration are shown in the following figure.

To change case details:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

      Note: Only members of the Super Admin group can change case details.

    2. On the Case Management ribbon, select Case Management > Case Management.

    3. In the Case Management window, select the needed managing client and case.

  2. Click the Case Details tab, then click the Modify Case button.

  3. In the Modify Case Details dialog box, change any of the available items. For details on these items, see Step 1: Set Up a Case.

    1. Code Name: Select from available ECNs.

    2. Matter number: Matter number is defined in conjunction with the ECN. If you change the matter number for one case, it changes for all cases having the same ECN.

    3. Managing client or client: Select from existing managing clients/clients.

    4. Data, index, and/or uploaded exhibits directories.

      IMPORTANT! If a directory already has content, change associated location(s) only if you copy the existing directory content to the new location(s).

    5. Native time zone offset

    6. Notes

    7. Tagging options

    8. Date and time format: Click Format Options to re-define the date/time format.

    9. Disable the case (see Enable or Disable a Case).

  4. When finished, click OK.

    Note: Users with appropriate permissions can view case details in Eclipse (web environment), where they are available from the Case Administration menu.

Import Case Objects

Case “objects” include configuration details for the following aspects of a case:

  • Tag palette

  • Redaction palette

  • Coding forms

  • Relationships

These objects can be imported from a case definition (.CSE) file to an existing case to speed case configuration.

Note: To import objects, the .CSE file must have been exported from Eclipse 2013 or later.

To import objects:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

      Note: Only members of the Super Admin group can import case objects.

    2. If the needed case definition file does not yet exist, it from an appropriate case. See Export a Case Definition File for details.

    3. On the Case Management ribbon, select Case Management > Case Management.

    4. In the Case Management window, select the needed managing client and case.

  2. Click the Case Details tab, then click Import Case Objects.

  3. Navigate to the .CSE file in the Select Case Field Definition File dialog box.

  4. Select the case file and click Open.

  5. In the Case Definition dialog box, review case object details; select the needed objects and click OK. Configuration details from the .CSE file will be added to the selected case.

Enable / Activate or Disable / Deactivate a Case

Note: You may use the terms “activate” or “deactivate”” a case interchangeably with “enable” or “disable” a case, respectively.

It may be required that you occasionally put a case “on hold” for some reason. Eclipse Administration allows you to disable /deactivate a case.

A disabled / deactivated case is not available to users in Eclipse or Eclipse Administration. It will be available to members of the Super Admin group.

To disable / deactivate (or enable / activate) a case:

  1. Get started:

  2. Start Eclipse Administration and log in as an Eclipse administrator.

  3. On the Case Management ribbon, select Case Management > Case Management.

  4. In the Case Management screen, select the required managing client and case.

  5. If you are disabling / deactivating the case and users might be working in the case, notify them to exit the case.

  6. Click the Case Details tab, then click Modify Case.

  7. To disable / deactivate the case, select the Mark Case as Disabled option (or clear the option to enable /activate the case).

  8. Click OK.

Delete a Case

This procedure permanently deletes a case. Back up the case if needed before you delete it (for precautionary reasons). To delete a case:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

      Note: Only members of the Super Admin group can delete a case.

    2. On the Case Management ribbon, select Case Management > Case Management.

    3. In the Case Management window, select the needed managing client and case.

  2. Make sure no one is working in the case (in Eclipse or another instance of Eclipse Administration). You cannot delete a case if any users are logged in to the case (or batches based on the case).

  3. In the Case Management window, make sure the correct managing client and case are selected in the left pane.

  4. Click the Delete Case button.

  5. In response to the warning message, click Yes, delete the case.
