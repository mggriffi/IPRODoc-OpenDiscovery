Import Users from File

Rather than creating users manually, you can import multiple users using a .CSV file.

The .CSV file must be in the following format:

true/false,UserName,LastName,FirstName,Password

where the first item in each row (“true” or “false”) identifies whether the user is enabled or disabled upon import. Disabled users cannot log in.

For example:

true,hbrown@myco.com,Brown,Henry,MyPassword@321

false,ssmith@myco.com,Smith,Sally,MyPassword@321

If needed, these details can be changed after the users are imported.

Note: Imported users are initially defined with the same managing client and optionally, user type (which you choose).

For large numbers of users who need to be defined with different managing clients and user types, the most efficient approach is to use separate files based on user type/managing client assignments so that these assignments can be made for all users at once, without the need for revisions after the import.

 

To import users from a .CSV file:

  1. Get started:

    1. Make sure you have a .CSV file with user details in the format explained above.

    2. Log in as a user who is a member of the Super Admin group.

    3. Click the Eclipse button, , and select Administration.

    4. If the System Administration pane is collapsed, click the pane title to expand it.

  2. Click Users.

  3. On the right side of the Users tab, click Import Users > From File.

  4. Click Open File, then navigate to and open the needed .CSV file.

  5. From the drop-down lists, select the needed Managing Client and optionally User Type. These selections will be applied to all selected users in the group.

  6. If a header row exists in your .CSV file, select the First line is column header option.

  7. Review the user details in the Import Users dialog box; if problems other than the Enabled settings exist, cancel the import, correct file errors, and then perform this procedure again.

  8. When the list is correct, if any of the users should be initially disabled, clear the Enabled option. Disabled users cannot log in to Eclipse.

  9. Click Import.

  10. While the import is in process, the list of users in the Users tab will be dimmed. When it becomes active, review the new users added to the list; revise details for individual users if needed.

  11. Inform users of their Eclipse log-in names and passwords.
