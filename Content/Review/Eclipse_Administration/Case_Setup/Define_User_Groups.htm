Define New User Groups

Cases are reviewed by users that are assigned to groups. Create groups that match employees to specific review needs.

  1. Get started:

    1. For best results, consider a naming convention for groups.

    2. Log in to Eclipse as a user who is a member of the Super Admin group.

    3. Click the Eclipse button, , and select Administration.

  2. If the System Administration pane is collapsed, click the pane title to expand it.

  3. Click Groups, then .

  4. Enter group details as described in the following table.

    5. Group Definition Details Table

    Item

    Description

    *Group Name

    Name for the group.

    NOTE: Duplicate names are allowed. If you have similar groups for different managing clients, consider a useful naming convention.

    *Managing Client

    Select the managing client this group is to be associated with. This selection determines which cases and users are available to be assigned to the group.

    If needed, click Create New Managing Client and define the client now. See Create Managing Clients and Clients for details.

    Notes

    Enter additional details if needed. (Notes are searchable in the Groups list, which can be useful.)

    Enabled

    Clear this option only if you do not want the group to be active at this time. Members of the group will not be allowed to log in to Eclipse.

    Case Assignments

    1. In the Case Assignments area, click Add/Remove Cases. Only cases assigned to the same managing client are available.

    2. Select the case(s) this group will review (and/or remove cases from the group). Note:

      1. To select all cases, click the option box next to Case Name.

      2. Scroll to view all cases and/or start typing a needed name in the search bar at the top of the list to find a specific case(s).

    3. Check your selections in the Selected Items area.

    4. When all cases are selected click Apply.

    TIP: A group can be assigned to a case from the case itself if the user has the Case Details permission.

    Group Users

    1. In the Group Users area, click Add/Remove Users. Only users assigned to the same managing client are available.

    2. Select the user(s) to be in this group (and/or remove users from the group). Note:

      1. To select all users, click the option box next to User Name.

      2. Scroll to view all users and/or start typing a needed name in the search bar at the top of the list to find a specific user(s).

      3. NOTE: Do not add more than 1,000 users to a group.

    3. Check your selections in the Selected Items area.

    4. When all users are selected, click Apply.

    * = Required

  5. Click Save.

View, Edit, or Delete User Groups

Note: Do not edit (other than to add or remove users) or delete the default SuperAdminGroup, MediaManager, or MediaManagerSelf- Service groups.

 

To view, modify, or delete user groups:

  1. Get started:

    1. On your site’s Eclipse website, log in as a user who is a member of the Super Admin group.

    2. Click the Eclipse button, , and select Administration.

    3. If the System Administration pane is collapsed, click the pane title to expand it.

  2. Click Groups.

  3. Review the list in the Groups tab. To find a specific group in a long list, see the tips in Use Lists.

  4. To view details of a particular group, click the name in the list; details appear on the right.

  5. To edit group details:

    1. Click the group in the list.

    3. Click Save.
