Migrate Cases from a Third-Party Database

Note: Eclipse Administration allows you to migrate a case from Concordance® only if Concordance is installed and you hold a valid license for it.

The following procedures assume you are familiar with Concordance and your existing cases.

 

The following content can be migrated from Concordance .DCB files[1]:

  • Database fields

  • Tags

  • Redaction categories

  • Issue tags and/or notes

  • Images: If using Concordance Image, the migration will convert your image base, including annotations such as redactions. The following annotations are not converted:

    Hollow Box

    Hollow Ellipse

    Solid Ellipse

    Line

[1]If you have Concordance .DAT files, see Import Case Data for details on importing the data. If you have Concordance .CAT files, migrate each file comprising the .CAT file individually into the same Eclipse case.

In addition to migrating the case, Eclipse Administration makes it easy to evaluate and change the Concordance case definition as part of the case migration. Thus the efficiency of migration is combined with the ease of customizing the case to meet your needs.

Migrating a case includes six steps, as outlined in the Migrate from Concordance window and discussed in this section

  • Step 1: Setting Up the Migration

  • Step 2: Mapping Fields

  • Step 3: Viewing Progress

  • Step 4: Viewing Imported Records

  • Step 5: Viewing Imported Tags

  • Step 6: Viewing Imported Redaction Categories

  • Step 7: Viewing Errors

Important: You cannot save a portion of a case migration (such as your entries for steps 1 and 2). If you exit Eclipse Administration before completing all four steps, you will lose your work.

Step 1: Setting Up the Migration

Preparation

Familiarize yourself with the Concordance database and review the Migrate from Concordance screens to determine how your Concordance data should be handled. If importing into an existing Eclipse case, review that case as well.

Add Basic Information

To start the migration process, add basic information for the case:

  1. Start Eclipse Administration and log in as a user who is a member of the Super Admin group.

  2. On the Case Management ribbon, select Case Management > Migrate Case > Migrate from Concordance.

  3. Click the Step 1 tab.

  4. Select Concordance Database: Click , then navigate to and select the Concordance database (.DCB file). Click OK.

    Note: Concatenated databases (.CAT files) should not be migrated. However, you can migrate individual .DCB files and/or import individual .DAT files for the databases comprising a .CAT file into one Eclipse case.

  5. To import into a current case:

    1. Click the Import into an Existing Case option.

    2. Select the needed managing client and case.

    3. Review field data on the right side of the tab.

    4. Skip to step 7.

  6. To create a new case from the migrated data:

    1. Click the Create a New Case option.

    2. Select the needed managing client.

    3. Complete the entries in the Create a New Case section. See the table in Step 1: Setting Up a Case for details.

  7. Continue with one of the following procedures:

    • If you are creating a new Eclipse case: Create/Map Fields for a New Case.

    • If you are migrating into an existing Eclipse case: Step 2: Mapping Fields.

Create/Map Fields for a New Case

If you are creating a new case, on the right side of the Step 1 tab:

  1. Review the Concordance fields.

  2. For each field you want to map, click .

  3. Complete the Edit Field dialog box to define the field. See Step 3: Defining Database Fields for details on selections.

  4. Click OK.

    Tip:

    • Ignore Concordance fields that should not be migrated.

    • To concatenate fields, ensure they are all defined as the same field type.

    • Identify the fields to be system fields and define them as FixedLengthText for now. (They will be mapped to system field types in the next step.)

  5. When fields are defined, ensure all system fields are mapped. If you skip this step, Eclipse will create the fields but they will not be mapped to any of the Concordance fields. See Step 3: Defining Database Fields for details on system fields.

    1. Click Map System Fields.

    2. If the names of needed fields match those of system names but have not been mapped, click Map by Field Name. Any names that match (regardless of capitalization) will be mapped.

    3. To map other fields, locate a system field that needs to be mapped in the Eclipse Field list, then double-click the corresponding Load File Field cell.

    4. From the field list, select the field that should be used as the system field.

    5. Repeat these steps for all other system fields to be mapped.

    6. When finished, click OK.

  6. When finished, continue to Step 2: Mapping Fields.

Step 2: Mapping Fields

If creating a new case

If you are creating a new case for the Concordance migration, use the Step 2 tab to review the fields created in Step 1 and complete migration choices:

  • If any field-mapping changes should be made, double-click the needed field in the Eclipse Fields column and select the correct field.

  • If any field(s) were defined in step 1 that should not be migrated, clear the Include option for the field(s).

If importing into an existing case

Complete one of the following methods (auto-select or manual) to map the fields in your Concordance case to the Eclipse case. Note:

  • Some fields in the Concordance database can be omitted (not mapped) if needed.

  • More than one field in the import file can be mapped to a single field in the case (concatenated). For example, the import file may have two fields to define an author’s name (First Name, Last Name). In the Eclipse case, a single field may be used for this information (Name).

  • If you will be concatenating fields, ensure that those fields are of the same type.

Auto-select fields

If you are importing into an existing case, Eclipse Administration can attempt to match the fields based on the field name. For example, if the Concordance case contains a field called IMGKEY and the case contains a field named IMGKEY, they will be automatically matched.

To auto-select fields:

  1. If you want to match by field name, ensure that your Concordance case includes field names in the header. In the Migrate from Concordance screen, click the Step 2. Map Fields tab.

  2. Click the Auto-Select By Field Name button.

  3. Evaluate the matching to determine if:

    • the auto-matching does not meet your needs,

    • some fields cannot be matched (in which case, the word “Select” in the Eclipse Field column will be red), and/or some fields should not be included.

  4. For fields that should not be included, clear the Include option.

  5. For other mapping problems, continue with Manually Match Fields below.

  6. When all fields to be included are mapped, continue with Complete Migration.

Manually Match Fields

To manually match the fields that have not been mapped or for which mapping needs to be corrected:

  1. On the Migrate from Concordance screen’s Step 2 tab, evaluate the Concordance Field listing and sample data to determine which fields will be included in the import.

  2. For each field to be migrated, ensure the Include option is selected.

  3. For each included field, if the corresponding Eclipse Field listed is not correct, click the Eclipse field and choose the needed field type (new cases) or field name (existing cases). The list shows the case’s field names and types to assist you with matching.

    Tip: Selecting a matching database field automatically selects the Include option for you.

  4. You will be alerted if you concatenate fields (match two or more fields to a single Eclipse field); see the following figure. If it is correct to have the fields concatenated, select a Merged Field Delimiter if needed (bottom right section of the screen). If selected, the delimiter will be included in the merged data to identify the original field delineation(s).

  5. Repeat these steps for all fields to be included.

  6. Continue with Complete Migration below.

Complete Migration

Complete the following steps to select additional options based on the content of the Concordance case being migrated and complete the migration:

  1. Ensure that the Step 2 tab is selected. The following options are at the bottom of this tab.

  2. Tag Options: If the Concordance file being migrated includes tag data, select from one of the following options:

    • Ignore Private Tags: Select this option to not import any private tags in the Concordance case. If this option is not selected, Concordance private tags will become Eclipse public tags.

    • Make Root Level Tag Groups: Select this option to import tags into a tag group named for the first level of a set of nested Concordance tags. If not selected, tags will be imported to the group “Imported Tags.”

  3. Issue Tags: If the Concordance file being migrated includes issue tags and you want to import them, select the Include Issue Tags option. In addition, select one of the following options:

    • Include in Same Group as Doc Tags: Select this option to include issue tags in tag groups defined for the case.

    • Create Issue Tag Group: Select this option to create a new group specifically for issue tags.

  4. Issue Notes: If the Concordance file being migrated includes issue notes and you want to import them, select the Include Issue Notes option. In addition, select one of the following options:

    • Import into Field: Select the field into which the notes will be imported.

    • Create New Field: Enter a name for a new field into which the notes will be imported.

  5. Options: Choose the following options as needed:

    • Merged Field Delimiter: If you have merged fields in Manually Match Fields, ensure that the correct delimiter is selected.

    • Date Format: Selected needed date option. If the Concordance date fields being migrated are true dates, they should be mapped to Eclipse DATE fields and the format should be Y/M/D. If dates are mapped to a text field, choose desired format.

  6. Index after process completes: Select this option to create the index used for text searching after the migration completes. To skip indexing now (for example, to save time), do not select the option. Indexes can be managed after data is imported as explained in Manage Case Search Indexes.

  7. When all options are selected, click Import Records. Your Concordance database will be imported into Eclipse.

  8. Continue with Step 3: Viewing Progress below.

Step 3: Viewing Progress

To assess the migration from Concordance:

  1. In the Migrate from Concordance screen, click the Step 3 tab.

  2. Review import details as they are presented.

  3. After the migration completes, click View to examine and/or save the import log.

  4. Take any of the following actions to evaluate migration results:

    • Step 4: Viewing Imported Records

    • Step 5: Viewing Imported Tags

    • Step 6: Viewing Imported Redaction Categories

    • Step 7: Viewing Errors

Step 4: Viewing Imported Records

Once data is migrated into Eclipse, complete these steps to view and verify record details:

  1. In the Migrate from Concordance screen, click the Step 4 tab.

  2. Possible activities include:

    • Reviewing the Documents list and document/page details

    • Deleting a document or page

    • Checking/changing image (or native) file path

    • Viewing images

Step 5: Viewing Imported Tags

Complete these steps to check tags in your migrated case:

  1. In the Migrate from Concordance screen, click the Step 5 tab.

  2. Review listed tags and make a note of any changes that might be required.

  3. When finished, continue with Steps6 and/or 7 (below).

Step 6: Viewing Imported Redaction Categories

Complete these steps to check redactions for your migrated case:

  1. In the Migrate from Concordance screen, click the Step 6 tab.

  2. Review listed redactions and make a note of any changes that might be required.

  3. Continue with Step 7 below.

Step 7: Viewing Errors

The Errors tab provides detailed information about migration errors. When errors exist, the tab name is red and the number of errors is indicated. To view errors:

  1. In the Migrate from Concordance screen, click the Errors tab.

  2. To view the first 25 entries in the error log file, click Display Errors. One line for each of the errors displays, indicating the type of error.

  3. To view specific errors relating to the load file, click Create Loadfile View. Records with errors will be listed with problem data listed in red.

  4. To save the load file, click Export and specify a location for the file. This file is named “FieldErrors” and will be a .DAT or .LFP file, depending on the type of errors. Use this file to correct load file errors, then re-import and overlay problem records.
