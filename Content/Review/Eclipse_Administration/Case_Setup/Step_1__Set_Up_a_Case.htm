Step 1: Set Up a Case

The first step in creating or enabling a case in Eclipse® by involves defining the basic case details:

  1. Log in as a user who is a member of the Super Admin group.

  2. On the Case Management ribbon, select Case Management > Create/Enable Case, then continue with step 3, step 4, or step 5.

  3. To enable a case created in eCapture® by :

    1. Select the needed case from the Case Name list. The following fields populate according to the eCapture definition of the case: Code Name (ECN), Managing Client and Client, and optionally an associated Matter Number. These fields cannot be changed during case creation. (If changes are needed, enable the case first, then make changes.)

    2. Complete the remainder of the case definition as described in the Set Up Case Tab Details table, then skip to Step 2: Load a Case Definition.

  4. To create a new case using an existing Code Name (optionally with matter number):

    1. Click  corresponding to the Case Name field.

    2. Enter a new case name.

    3. Select the needed Code Name. This selection also determines Matter Number.

      Note: If the selected Code Name is not associated with the respective Managing Client or Client, you will not be able to create the case until all such inconsistencies are resolved.

  5. Complete the remainder of the case definition as described in the "Set Up Case Tab Details" table below, then continue to Step 2: Load a Case Definition.

  6. To create both a new case and a new Code Name (optionally with matter number):

  7. Click  corresponding to the Case Name field.

  8. Enter a new case name or go to step c to make the case name the same as the ECN.

  9. Click  corresponding to the Code Name field and enter a new, unique ECN of up to 256 Unicode characters.

  10. To use the ECN for the case name, select the Same as Code Name option.

  11. Complete the remainder of the case definition as described in the following table, then continue to Step 2: Load a Case Definition.

Set Up Case Tab Details

Item

Description

-level Details

Required when creating a new case; provided when enabling an eCapture case.

Code Name (ECN)

This name (also called “Code Name”) serves as a primary organizer for a set of related cases, as a shortened version of a very long case name, or for any purpose that will help you organize and recognize cases/projects across all products.

The ECN is managed in the database and available to all products. It can be created in any product.

When creating a new case in Eclipse, you must either select an existing ECN or create a new one (maximum of 256 Unicode characters).

Matter Number

Optional: The matter number is tied directly to the ECN. If you create a new ECN, you may include a matter number (maximum of 128 Unicode characters).

Case Name

When creating a new case in Eclipse, you must enter a case name. This name is managed in the database and available to all products.

Case names must be unique for a selected managing client and can be 256 Unicode characters maximum. The following characters are not allowed: * . " / \ [ ] ; : | = ' ? > <

Managing Client

The managing client coordinates available user groups with cases and review batches.

When creating a new case in Eclipse, select the needed managing client. If it is not listed, add it as explained in Create Managing Clients and Clients, then return to creating cases. (May also be created in or eCapture.)

Client

A secondary or subordinate client specific to the selected managing client.

When creating a new case in Eclipse, select the needed client. If it is not listed, add it as explained in Create Managing Clients and Clients, then return to creating cases. (May also be created in or eCapture.)

Eclipse Details

Required whether creating a new case or enabling an eCapture case.

Data Directory

A location where Eclipse will place database-specific files and other files required for the case. The data directory is required; a case-specific folder is created here for each case.

Enter (or click  to navigate to/select) the needed location.

The location must be a UNC path and must be accessible by the dtSearch function. This is typically the same network location on which dtSearch is installed, and/or the network location where case image files are located.[1]

Index Directory

The location for dtSearch indexes. Note:

Enter (or click  to navigate to/select) a location where Eclipse will place dtSearch indexes. The locations must be accessible by the dtSearch function and entered in UNC format.

Select the Same as Data Directory option to place items in the main Eclipse data location.

You may want this directory in a location different from the data directory to speed up processing, address space issues, or for other reasons.

Uploaded Exhibits Path

The location for documents linked to transcripts. Note:

Enter (or click  to navigate to/select) a location where Eclipse will place transcript-related documents. The location must be entered in UNC format.

Select the Same as Data Directory option to place items in the main Eclipse data location.

You may want this directory in a location different from the data directory to speed up processing, address space issues, or for other reasons.

Number of indexes

The default dtSearch index is the index used by default, and ten index files are defined by default. Change this number if needed.

Particularly for large and diverse data collections, it can speed processing to allow multiple index files.

Native Time Zone Offset

Optional: This setting controls the offset to Greenwich Mean Time (GMT) that is applied during date and time formatting. The default setting is GMT (0 offset).

Click  and select a time zone setting to control how header date and time values display for emails in the Eclipse Native Viewer tab. The setting also controls the date and time for TIFF images generated in the Eclipse Native Viewer tab (TIFF-on-the-Fly). This setting will not affect or modify dates and times stored as text or in the metadata fields associated with the displayed values. Thus, the values in the native files may differ from those in the Native View tab (or TIFF-on-the-Fly images).

Note: An offset may also be defined in eCapture. If your case documents will come from eCapture, you may want to check with the eCapture administrator and use the same offset.

Notes

Optional: Enter up to 500 characters of pertinent details.

[1]If dtSearch is installed on your personal computer, the Data Directory can be located anywhere you specify on the local drive. For example, \\NW01\Ipro Eclipse\ Case Directories.
