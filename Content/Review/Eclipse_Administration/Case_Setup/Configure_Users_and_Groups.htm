Users

Users are individual members of your organization who will use Eclipse (for example, case reviewers, attorneys, or other specialists).

Ensure that your Eclipse license is sufficient for the number of users you want to add. If you have any questions about the number of seats available, contact IPRO Support.

Note: Starting with release 2015.2.0, Named User Tracking is available for Eclipse. The “named user” model allows product billing to be based on the number of users active during each month, rather than on a maximum number of named users.

Contact your Sales Representative to learn the benefits of this capability or Support for technical assistance.

User permissions are determined by group and case assignment.

Starting with release 2014.1.0, (Services component) serves as a centralized database in which all users, including Eclipse users, are managed. Users can be created in System Administration or Eclipse and once enabled, they are available in all applications.

User Types

Eclipse users can be identified by type, such as “Billable” and “Non-billable.” You can create any user types needed to track activities at your company.

Groups

Users are combined into groups that are assigned to cases for review and management purposes. Permissions are assigned to groups in conjunction with case assignment, allowing secure case reviews and efficient case management. Each group can have up to 1,000 users.

Three default groups exist in Eclipse:

  • SuperAdminGroup: This group is for administrators who govern all aspects of Eclipse at their site. Members of this group can perform all actions for all cases within Eclipse and Eclipse Administration, as well as all actions in . Users in this group must be added in Eclipse.

  • MediaManager: Users in this group are equivalent to Eclipse Super Administrators, plus they can access and perform all actions (Administration, Media Management, and Self-Service).

  • MediaManagerSelfService: This group is intended for non-administrative users who will be uploading files through Self-Service. Users in this group have access only to the stand-alone Self-Service page (unless the group is assigned to one or more cases in Eclipse).
