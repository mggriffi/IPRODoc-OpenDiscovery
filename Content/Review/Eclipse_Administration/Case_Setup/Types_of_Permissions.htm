Types of Permissions

Permissions govern activities in both Eclipse and Eclipse Administration; they are grouped basically as described in the following paragraphs.

Permissions are grouped as follows:

  • System Administration Permissions

  • User Interface Permissions

  • Security Permissions

  • Mass Actions

  • Eclipse Administration Permissions

  • Security Beyond User and Group Permissions

System Administration Permissions

Members of the Super Admin group can perform all actions within Eclipse and Eclipse Administration.

Members of this group are the only users with access to the System Administration pane in Eclipse. Here, the security components listed can be managed.

The ability to perform certain Eclipse Administration functions are also limited to this group.

Security Permissions

These permissions govern user actions in Eclipse. Rights and options are assigned to groups; users in the group will have the ability to perform associated tasks in the cases to which the group is assigned.

Security Permissions

Permission

Allow users to...

Documents

  • Rights: View, Edit

  • Options:

    • Print

    • Create Tiff on the Fly

    • Delete Image

    • Launch Native Files

    • Modify Grid Display

    • Quick Search

    • Add New Pick List Items

View and/or edit records.

The Edit right allows editing in Record View.

Options govern the document-specific actions listed; other permissions may also be required (such as right to view the Image tab).

Tag Palette

  • Rights: View, Edit, Create

  • Option: Page Tags

View the Doc Tags tab (including tags that have been applied to documents), and edit or create tags (in Eclipse Case Administration or during document review).

Optionally, have access to the Page Tags palette.

Note: To apply/remove tags to/from documents, users must also have the right to edit documents.

Annotations

  • Rights: View, Edit, Create, Delete

  • Option: Modify/Delete other users’ annotations

View, edit, create, or delete mark-ups, sticky notes, embedded text, or highlighting. Optionally modify/delete annotations made by other users.

Note: To work with annotations, users must also have the right to view the Image tab.

Users with the rights to view annotations will have the option to print them if they have the Documents Print permission.

Redactions

  • Rights: View, Edit, Create, Delete

  • Option: Modify/Delete other users’ redactions

View, edit, create, or delete redactions. Optionally modify/delete redactions made by other users.

Note: To work with redactions, users must also have the right to view the Image tab.

Users without the right to view redactions will see all document content if they have permission to view Documents.

Saved Searches

  • Rights: View, Edit, Create, Delete

View and run (1), edit, create, or delete saved searches.

Note: To work with saved searches, users must also have the right to view Documents and Advanced Search.

Shared Folders

  • Rights: View, Modify

View and/or modify a search folder that is shared. Folders are shared based on a user’s group assignment.

See the Saved Searches note above.

Smart Folders

  • Rights: View, Build, Edit, Create, Delete

View, rebuild, edit, create, or delete Smart Folders.

See the Saved Searches note above.

Expand Batch List

  • Rights: View, Edit, Split, Delete

View, edit, split, or delete batches in the Batching tab.

Build Categories

Gives users the ability to build the categorization index in Eclipse Analytics (on the web).

The View permission allows users to create/run advanced searches as well as view and run public saved searches.

 

Mass Actions

Governs “global” actions (actions performed on multiple documents at one time).

Mass Action Permissions

Permission

Allow users to...

Print

Print selected (multiple) documents at one time.

Edit

Apply a tag(s) to multiple documents at one time or change field content for multiple documents at one time.

Note: This permission is also required for tagging search results from the Doc and/or Page Tags tabs.

Export

Export data for selected documents to a .CSV or .DAT file.

 

Eclipse Administration Permissions

Eclipse Admin App permissions give users access to the following functions in Eclipse Administration:

Eclipse Administration Permissions

Permission

Allow users to...

Case Management

Create or migrate cases and manage such case components as fields, reports, and views.

Also governs ability to perform such actions as transcript import and configuration, global replace, path validation, and Eclipse Analytics management.

Export Case Data

Export data from Eclipse Administration.

Import Case Data

Import case data and documents.

 

Security Beyond User and Group Permissions

In addition to the permissions assigned in Eclipse System or Case Management, security is set in Eclipse Administration as follows:

Security Beyond User and Group Permissions

Permission

Allow users to...

Alerts

Individual alert messages.

Batches

Access to batches of a case to which the user group is assigned.

Document Views

Use of different document views.

Fields

Individual fields can be secured (view-only, update, and/or denied) for cases or user groups.

Fields

Access to individual reports.
