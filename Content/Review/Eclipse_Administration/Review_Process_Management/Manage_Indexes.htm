Change Index Order

The order of indexes in the Manage Indexes screen is the order they will be listed for users in the Advanced Search screen. To change the order of indexes in the list:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client/case.

  2. In the index list, click an index to be moved and click Up or Down as needed.

  3. Repeat to move other indexes.

Edit an Index Profile

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client/case.

  2. Click Manage Index Profiles.

  3. In the Profile Name list at the top of the dialog box, select the profile to be changed.

  4. Make changes to the basic profile settings. See Create an Index Profile for details.

  5. Edit filters if needed:

    1. To edit an existing filter, click  corresponding to the filter and make needed changes. See step 5 under Create an Index Profile.

    2. To delete an existing filter, click  corresponding to the filter.

      Tip: A confirmation message does not appear; if you delete the wrong filter, close click Close Window without saving modifications.

  6. When finished, click Save, then Close Window.

Change an Index Definition

An index definition can be changed while users are working in Eclipse. To change index basics:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client and case.

  2. In the index list, click  corresponding to the index to be edited. (You may need to scroll horizontally to find this button.)

  3. Change the index details, then click Save.

Add Documents to an Existing Index

Documents can be added to be added to an index by selecting a search. To process additional documents:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client and case.

  2. In the index list, click (Button) corresponding to the index which is to have documents added (You may need to scroll horizontally to find this button.)

  3. A dialogue will open prompting you to choose from a list of searches. Select the one with documents you wish to add to your index.

  4. Click OK to process the new documents.

    Note: If there are no new documents to be processed in your selected search, you will be prompted to select one with unprocessed documents before you are allowed to proceed.

Rebuild an Index

Indexes may need to be rebuilt if data is added to or removed from a case or the index training set, or for other reasons.

Whether or not indexes can be rebuilt while Eclipse is being used and users are performing searches involves options selected as part of the index profile definition.

To rebuild an Index

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client and case.

  2. If your index profile allows concurrent index updates, skip to step 5. If the profile does not allow concurrent updates, notify users that the index will be rebuilt and offer instructions for continuing work, such as using an alternate index, until the index is rebuilt.

  3. Select the index to be rebuilt and click Disable Queries; Notify users that the index will be unavailable while the index is being rebuilt.

  4. With the index still selected, click Build to completely re-index data or Incremental Build to rebuild the index based only on changes since the last build.

  5. When the build completes, click Enable Queries. Optionally, select the Enable Queries on Startup option.

  6. Click View Log File to evaluate any issues with the build process.

  7. Notify users of the changes and if needed, provide instructions on correct use of the rebuilt index.

Inactivate an Index

An inactive index is not available for use in Eclipse.

To make an index inactive:

  1. Notify users that the index will be removed and offer instructions for continued work, such as using an alternate index.

  2. Start Eclipse Administration and log in as an Eclipse administrator.

  3. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

  4. Select the needed managing client and case.

  5. In the index list, click  for the index to be inactivated. (You may need to scroll horizontally to find this button.)

  6. Clear the Active option and click Save.

    Note: To improve system performance, inactivate Eclipse Analytics indexes that are not needed for current review activities.

Disable User Queries

To disable queries:

  1. Notify users that the index will be removed and offer instructions for continued work, such as using an alternate index.

  2. Start Eclipse Administration and log in as an Eclipse administrator.

  3. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

  4. Select the needed managing client and case.

  5. In the index list, click the index for which queries are to be disabled.

  6. Click Disable Queries.

Delete an Index Profile

A profile can be deleted if it is not associated with any index.

To do so:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes screen, select the needed managing client and case.

  2. If the profile to be deleted is associated with an index, select a different profile for the index. See Edit an Index Definition.

  3. Click Manage Index Profiles.

  4. Select the profile in the Profile Name list.

  5. Click Delete.

  6. In response to the confirmation message, click OK.

  7. Click Close Window.

Delete an Index

To delete an index:

  1. Get started:

    1. Notify users that the index will be removed and offer instructions for continued work, such as using an alternate index.

    2. Start Eclipse Administration and log in as an Eclipse administrator.

    3. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    4. Select the needed managing client and case.

  2. In the indexes list, click  corresponding to the index to be deleted. (You may need to scroll horizontally to find this button.)

  3. Click OK in response to the confirmation message.
