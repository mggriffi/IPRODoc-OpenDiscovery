Review Pass Basics

Review passes are based on searches saved in Eclipse. Thus, searches must be planned and saved in Eclipse before you create review passes. For each review pass, you will define:

  • The review pass name and the saved search on which it is based.

  • The maximum size of each batch.

  • A prefix for batch numbering.

  • The search upon which the review pass will be based.

  • Optionally, you may define:

    • A batch priority (Low - Critical).

    • A category to define how batches are grouped

    • A field indicating family relationships.

    • A sort order for fields in the case.

  • Finally, you may limit the review pass to specific user groups.

Create Review Passes

To create a review pass:

  1. Get Started:

    1. Complete planning as explained in Before You Begin. Start Eclipse and log in as an Eclipse administrator.

    2. Log in to Eclipse and open the case for which review passes are being created.

    3. In the Case View pane, click Case Administration, and then Review Pass Management as shown in this figure.

  2. Click .

  3. Enter a name for the review pass and complete other required entries:

    1. Enter the maximum Batch Size (number of documents) that each batch can be. This entry must be ten (10) or greater.

    2. Enter a Batch Prefix for batch names. This name is added to sequential numbering for the batches.

    3. Select the Saved Search upon which the review pass will be based.

  4. As needed, complete optional entries described in the following steps.

  5. Optional: Select a Layout Style. The default setting is Grid Review. If Grid Review is selected, the grid displays when the review pass loads along with the document view tabs window in the foreground. If the Form Review is selected, the document view tabs appear when the review pass loads.

  6. Optional: Select a Batch Priority to be assigned to all batches in the set.

  7. Optional: In the Batch by Category field, select a method for grouping documents within batches.

  8. Optional: Select a Family Field to keep related documents together.

    For steps 6 and 7, see How Documents are Organized into Batches to determine whether you want to set either of these options.

  9. Optional: Define a batching (sort) order based on up to three fields.

  10. Optional: Click  and select the user groups who will be reviewing the documents in this review pass. If specific groups are not assigned to the review pass, then all users in all user groups assigned to the case will be able to select batches in the review pass.

  11. When all entries are complete, click Save.

  12. Go to the next procedure to continue defining the review pass.

Create Batches

To create batches:

  1. Get started:

    1. Start Eclipse and log in as an Eclipse administrator.

    2. Open the case for which review passes are being created.

    3. In the Case View pane, click Case Administration, and then Review Pass Management.

  2. In the left pane of the Case Administration tab, click the needed review pass. If a large number of review passes exists, take any of the following actions to find the needed one:

    1. Scroll vertically and/or horizontally.

    2. Drag the pane’s borders to make it wider.

    3. Sort the keyword list names by clicking the sort button, , and choosing from the different options. Click a selected option again to reverse the sort order.

    4. In the search box above the left pane, start typing any of the following items: the keyword list name, the name of the administrator who created/modified the group, or part of the created/modified date. This is a plain text search that searches these items in order.

  3. Click  to update data in the Review Pass Information area.

  4. Review the number of unbatched documents in the review pass, then click Create Batches.

  5. After batches are created, evaluate data in the Review Pass Information area. Click  again to ensure data is updated.

  6. Repeat these steps to create other review passes.

  7. When finished, provide instructions to users who will be reviewing the documents in the batches. For details on providing instructions within Eclipse, see Add Case Instructions, Documents, and Alerts.
