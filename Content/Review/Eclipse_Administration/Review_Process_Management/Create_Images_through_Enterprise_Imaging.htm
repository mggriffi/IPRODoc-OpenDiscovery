Create Images through Imaging

Understanding Imaging

Starting with version 2015.0.0, Eclipse offers Imaging, an optional feature that allows Eclipse to work seamlessly with eCapture to address the issue of missing or damaged image files in your Eclipse case. Requirements include:

  • An Imaging license is required.

  • The Eclipse case must exist and needed fields must be defined.

  • eCapture 2015.0.0 or later must be installed and an eCapture project must exist for your Eclipse work.

  • The Eclipse Scheduler Service and Agent must be installed.

  • Native files must exist for the missing or damaged image files.

Permissions Required

All members of the Super Admin group can run Imaging. Others who have the Imaging permission can also perform this task. For purposes of images, "Eclipse administrator” refers to either type of user.

Preparation

In addition to the requirements listed above, you must define and save a search to identify the documents to be processed. For example, you may define/save a search for all documents for which the HASIMAGE field equals No and the HASNATIVE field equals Yes.

You should also review the imaging options available in Eclipse and those of your eCapture project to determine how to configure the Eclipse imaging job. See the table in Define and Run an Imaging Job for details on Eclipse imaging options.
