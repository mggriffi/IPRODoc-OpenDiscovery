Create an Index

To create an index for conceptual searching:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

    3. In the Manage Indexes window, select the needed managing client and case.

    4. If saved searches are needed for the training and/or searchable sets, make sure these searches have been saved in Eclipse. (See steps 6 and 7.)

  2. Click Create New Index and complete the dialog box as explained in the following steps.

  3. Name: Enter a meaningful, unique name for the index.

  4. Configuration Profile: Select the needed profile. If the needed profile does not exist, click New and define the profile as explained in Create an Index Profile.

  5. Active: Select this option unless you do not want the index to be available for searching until a later date.

  6. Training Set: Select the training data for the searchable index, either the entire case or the search-result documents for a public search saved in Eclipse. Training items should be representative of all searchable items. Note:

    • Using the entire case as training data requires more memory than using a subset of the searchable items.

    • Training datasets can be much smaller than the entire case as long as the concepts and terms contained in the case are well-represented in the training items.

    • At least ten training data items are needed to build an index; more than ten are preferred. If you provide fewer than ten training data items for building an index, the build process will fail.

  7. Searchable Set: Select the set of documents that will be searched when the search is executed in Eclipse. This may be the entire case or the search-result documents for a public search saved in Eclipse.

  8. Stop Words: To use a default list of over 600 stop words (commonly used words that should be ignored, such as in, or, of) for the index, skip this entry. Or complete step 9 or step 10 to customize the list.

  9. To create a custom list of stop words, enter or paste the list of words into the Content Analyst Stop Words dialog box, then click OK. Ensure only one word is on each line.

  10. To modify the default list of stop words:

    1. Click .

    2. Click Load Default Stop Words.

    3. Add words to or remove them from the list using common editing methods. Ensure only one word is on each line.

    4. When finished, click OK.

  11. Click Save.
