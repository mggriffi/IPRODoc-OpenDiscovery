Import through Integration

Eclipse® by can receive case data and image, native, and/or text files directly from eCapture® by , or starting with release 2015.2.0, through Streaming Discovery.

Also starting with release 2015.2.0, Eclipse supports concurrent “Direct to Eclipse” imports into a single case from either eCapture (or Allegro, if it exists at your site.) or Streaming Discovery. The amount of imports that can be handled at once depends on the number of available Eclipse Agents.

If the number of agents is not sufficient to handle all jobs at once, then they are added to a queue in the Schedule Manager and processed in order. Eclipse Administration displays a “Syncing with Scheduler” message if a job is placed in the queue.

Note: Eclipse activities that may be managed concurrently include:

  • Imports through integration

  • Migrations from Concordance

  • OCR imports

  • Eclipse Analytics email threading and/or near duplication (“dupe”) detection

Requirements include:

  • The Integration Service must be installed and the service running.

  • The Eclipse Scheduler Service and Agent(s) must be installed and the service running.

  • The Eclipse case must exist and needed fields must be defined.

  • Product versions must be:

    • Eclipse 2014.0.0 or later[1]

    • eCapture 6.0.0 or later[2]

    • To use Streaming Discovery, a Streaming Discovery worker license is required, and Eclipse and eCapture (and optionally Allegro) version 2015.2.0 or later are required.

[1]Earlier versions of Eclipse accepted eCapture (or Allegro) data using a different methodology.

[2]Or Allegro 3.0.0 or later, if it exists at your site.

About Integration

In all cases, a “Direct to Eclipse” export job is created and processed. During processing, one or more .DLF load files are created, which are then imported by Eclipse into an existing case (selected in eCapture or Allegro). Note:

  • In eCapture, the data set that is imported is based on one or more eCapture processing jobs and additional user selections.

  • For Streaming Discovery, the data set that is imported is based on a selected export series and additional user selections.

  • In Allegro, the data set that is imported is based on one or more “snapshots,” which are data sets filtered in a certain way (for example, by a date range), as well as other user selections.

Monitor Processing

After jobs are sent from eCapture or Streaming Discovery (or Allegro) to Eclipse, they can be monitored in Eclipse as follows:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. Click the Eclipse button, , and select Administration.

  2. In the System Administration pane, click Schedule Manager.

    Note: You may need to wait for jobs sent from eCapture (or Allegro) to display in the Eclipse Schedule Manager. The amount of time depends on job size and complexity.

  3. Whenever viewing the Schedule Manager tab, click Refresh periodically in the Job Actions pane to ensure details are up to date; complete of the following steps as needed.

  4. Review jobs and statuses: Review the list of active jobs in the left pane for an overview of all scheduler activity. Jobs are listed by case name; eCapture (and Allegro) jobs include the identifier “DLF Import.”

    Click next to a job name to see the individual tasks and statuses. Each job includes multiple tasks (“child jobs”), such as staging, integration, and indexing. The jobs list includes color-coding to indicate job and task status:

     

    Status

    Job or TAsks Is...

    On Hold

    Job/task is waiting to be placed in worker queue.

    In Queue

    Job/task is waiting to begin.

    In Process

    Job/task is being processed.

    Complete

    Job/task has successfully completed.

    Failed

    Job/task has been stopped due to an error.

  5. Find a job/specific details: Take any of the following actions to find a specific job or job details:

    1. Click Refresh in the Job Actions pane.

    2. Scroll vertically and/or horizontally in the jobs list.

    3. Click the right border and drag to make it wider.

    4. Sort the list by clicking the sort button, , and choosing from the different options. Click a selected option again to reverse the sort order.

    5. To find specific details in a long list, start typing any of the following items in the search box: the case or agent name, job type, part of the job’s scheduled or completion date, or job status. This is a plain text search that searches these items in order for all jobs (not tasks); the list is filtered as search matches are located.

  6. View job/task details: Expand a job and click a specific task to view details for the task; click Refresh periodically to ensure details are up to date. Note:

    • When a job is in progress, details and errors for each task display in the Job Details pane when the task is selected.

    • Completed jobs are routinely removed from the list.

    • If a job fails, it remains in the list. Its error log can be saved and the job can be restarted as described in the next procedure.

  7. Evaluate error files: An exportfile_errors.dat file may be generated during imports, where exportfile is the name of the original load file. This file includes field-level error details.

    This file is placed in the export folder containing the .DLF files created at the beginning of the export from eCapture (or Allegro).

    After evaluating the error details, make needed corrections and import the corrected .DAT file in Eclipse Administration.

Restart an Job

If an eCapture or Allegro job fails, take the following steps:

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. Click the Eclipse button, , and select Administration.

  2. In the System Administration pane, click Schedule Manager.

  3. In the jobs list, expand the failed job and click the task that has failed.

  4. Review data in the Job Details pane and/or save details to a separate text file by clicking Save Log and completing the Save As dialog box.

  5. Analyze the log and assess other possible issues (such as system resource problems) to determine the cause of the failure.

  6. Make needed corrections.

  7. After corrections have been made:

    1. Return to the Schedule Manager.

    2. In the jobs list, locate and click the failed task.

    3. Click Reset Job.

  8. Monitor the job as described in the previous procedure.
