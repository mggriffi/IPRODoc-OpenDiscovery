Manage Case Reports

Eclipse includes a variety of reports to assist with review tasks, administrative duties, and case oversight. Reports are presented in an easy-to-use table format.

Case Reports

The following table lists case reports—note that some reports can be configured in Eclipse Administration. (Security can be set for any/all of the reports.):

Administrator Configuration

Case Data Summary

Summarizes case size details such as number and total size (in GB) of files (image, native, and text files).

Users have the option to generate the report based on any field for which the Allow Group By option is set to Yes.

For example, if the BEGATTACH field is defined with Allow Group By, a report could be generated to summarize size details for all documents with attachments.

Gap Report

Identifies missing pages, that is, breaks in the numbering sequence of image keys (case, production, or other document numbers) for a particular case. For example, if ECL000009 is a single-page document and the next document number in the case is ECL000012, a gap exists and will be included in the report.

 

Privileged Report

Identifies documents that have been tagged as defined for the report. Included are document totals, the applied tag(s), and other items if defined.

Fields, tag groups, and/or tags can be added to the report if needed.

Production Summary

Summarizes production jobs completed in Eclipse Administration.

Summarizes redaction activity for a selected case.

 

Redaction Detail

Includes detailed information on redacted documents and other items if defined.

Fields, tag groups, and/or tags can be added to the report if needed.

Review Audit

Identifies inconsistencies in the tagging of related documents (as identified by a field defined with the Allow Group By option).

 

Review Status by Custodian

Presents review status of documents in a particular review pass, sorted by a field that contains custodian details. Tag details may also be included if configured.

X

Review Status by Reviewer

Includes review status of documents in a particular batch, sorted by assigned reviewers and tags if defined.

Tags can be added to the report if needed.

X

Reviewer Edits per Day

For a specific date range, lists all reviewers by user name and the number of distinct actions (field editing and/or tagging) each reviewer has made on each day of the date range. Can be useful in evaluating review efficiency.

 

Reviewer History

Summarizes reviewer activities for a selected period of time, including total hours spent by each reviewer, number of edits, and edits per hour and day.[2]

 

Reviewer History Detail

Provides detailed information on reviewer activities for a selected period of time, including day-by-day summary of time spent and editing details.

 

Summary by Custodian

Summarizes review activities (number of documents reviewed, not reviewed, etc.) sorted by a field selected by your administrator that contains custodian details.

 

TAR Reports

If you have implemented Technology-Assisted Reviews (TAR) at your site, associated reports are created as part of the TAR process. See Evaluate TAR Reports for TAR report details.

 

Transcript Reports

Generate reports on transcript annotations or searches.

 

[1]Configuration is regarding which fields have the Allow Group By option set to Yes.

[2]Editing includes tagging and field entries or changes.

Configure Standard Case Reports

Define or modify the content of eligible reports (listed in the previous table) as follows. If desired, open Eclipse and review the default contents of the reports before you modify them.

  1. Get started:

    1. Start Eclipse Administration and log in as an Eclipse administrator.

    2. On the Case Management ribbon, select Case Management > Case Management.

    3. In the Case Management screen, select the needed managing client and case.

  2. Click the Reports tab. (If it is not visible, click  on the right side of the window’s tabs.)

  3. Click  next to a report to be edited and continue with the following steps.

  4. To add content to the report:

    1. In the Edit Report dialog box, locate and select a needed item(s) (field, tag, and/or tag group as applicable) in the Available (left) list. Tips:

      • Scroll to see all available items.

      • To select multiple items, click an item name, then Shift+click or Ctrl+click additional items to choose a contiguous or non-contiguous set, respectively.

    2. When items have been selected, click .

    3. Repeat these steps if needed to add other items.

  5. To remove content from a report:

    1. In the Edit Report dialog box, locate and select an item to be removed (field, tag, and/or tag group as applicable) in the Selected (right) list.

      Scroll to see all selected items.

      To select multiple items, click an item name, then Shift+click or Ctrl+click additional items to choose a contiguous or non-contiguous set, respectively.

    2. When items have been selected, click .

    3. Repeat these steps if needed to remove other items.

  6. When items to appear in the report have been finalized in the Selected column, organize them in a way to best meet the needs of your users. Items will be displayed left to right in Eclipse based on the top-to-bottom order specified here. To change the order:

    1. In the Selected list, click an item to be reordered.

    2. Click Up or Down until the item is in the desired location.

    3. Repeat these steps to reorder other items.

  7. When finished, click OK.

  8. Repeat these steps for other reports to be configured.

  9. Optional: Continue with report security (next procedure).
