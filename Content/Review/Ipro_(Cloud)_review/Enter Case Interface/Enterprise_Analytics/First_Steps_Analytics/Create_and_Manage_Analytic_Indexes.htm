Create and Manage Analytic Indexes

Prior to creating an Analytic Index, a Configuration must exist. There are two types of configurations available, TAR and (AL). The configuration is created automatically when an AL Review pass is created. You can also create a custom AL configuration if you do not want to use the default. For more information on TAR or , see Manage TAR Projects and Manage Review Passes.

In this topic we will discuss creating an Analytic Index based on a TAR configuration. This type of Index must be created prior to using the Similar Documents relationship option and the Concept Search option in Advanced Search.

  1. On the Case Management ribbon, select Eclipse Analytics > Manage Analytics Indexes.

  2. Select the needed managing client and case.

Create a TAR Configuration

  1. Click the button to open the dialog window.

  2. Select <new> Configuration.

  3. Enter the appropriate information in the following fields:

    Item

    description

    Name

    Enter a name for the configuration profile.

    Index Type

    Select TAR Index (Active Learning is for Batch Reviews only)

    Similarity Function

    This option is for Near-Duplicates only

    Text Assumption Level

    Select one of the following options to determine how documents will be treated when analyzing for non-binary documents (i.e. documents that contain readable text as opposed to just numbers):

    • 0: Only treats documents as non-binary after the system has analyzed the text and identified the content as containing prose.

    • 5: If the document was converted from a known format to plain text, it assumes that the result should be treated as non-binary.

    • 10: In addition to the assumptions made at level 5, if the document is in a format that can’t be translated into text, it assumes that the document can be treated as non-binary without further translation of validation.

    Similarity Cutoff

    Enter the Similarity Cutoff value (from 1 to 100) to be used for determining the threshold for how similar two documents can be. The default value is 25.

    Insignificant Words

    Default list of stop words (commonly used words that should be ignored, such as in, or, of). Words may be added or removed from this list.

    Significant Words

    Add words that you want to ensure the index pays attention to and are known to be important for determining relevance, or words that you might want to include in analytics searching.

Create an Analytics Index

When choosing a subset of documents to be indexed, keep in mind that the indexing process has several criteria that are automatically applied. This helps to ensure that the documents submitted for indexing will provide the most valuable results:

  • Content included in your content filters are automatically excluded from the index.
  • Documents automatically identified as Short or Binary during indexing will be listed as such in the IA Doc Treatment field and are automatically excluded from the index.
  • Extremely large text files (> 4MB) are not automatically excluded, but if there is an abundance, it may result in the index building more slowly. You might want to consider excluding these from your search criteria if they are in question.

 

  1. Click .

    The Create New Analytics Index dialog window appears.

  2. Enter a name for the Index profile.

  3. Select your documents to Index

    1. To index all documents in the case, select <Entire Case>.

    2. To index a subset of documents based on a saved search, select the search and click OK.

  4. Select your Configuration.

    Note: The TAR configuration is used within Review to search for similar documents and concept searching. Active Learning configurations are for Review Passes only.

  5. Click the checkbox to apply content filters and click OK. See Create and Manage Content Filters for more information on content filters.

Manage Analytics Indexes

Edit or Delete an index

  1. To edit an existing index, click corresponding to the necessary index.

  2. In the Edit Analytics Index dialog box, make any needed changes in the following fields:

    item

    description

    Index Name

    Edit the name of the index profile.

    Documents to Add

    Documents can be added to an index by selecting one of the following options:

    • To add all documents in the case to the index, select <Entire Case>.

    • To add a subset of documents based on a saved search (including private searches), select the search and click OK.

    Note: If there are no new documents to be processed in your selected search, you will be prompted to select one with unprocessed documents before you are allowed to proceed.

  3. To delete an existing index, click corresponding to the index you wish to remove from your case.
