Create Batches for TAR Review Passes

The review passes created for TAR projects include the following, where ABC is the prefix defined for the TAR project. Create batches for these review passes at the times listed. See Workflow Process for details.

  • TAR_ABC_1. Training: create the training batches after the TAR project is created. Training batches contain the prefix “TAR_ABC_TR.”

  • TAR_ABC_2. Validation: create a single validation batch for each validation round after the training phase is complete and TAR categories have been rebuilt.

    • Validation batches contain the prefix “TAR_ABC_VR.” For each validation round, the batch name increments by one.

  • TAR_ABC_3. Certification: create a single certification batch after the validation round is complete and TAR categories have been rebuilt. The certification batch contains the prefix “TAR_ABC_CR.”

    • Note:

    • Do not change the default batch size for the validation or certification rounds. Create only one batch each for these rounds.

    • Do not apply security to TAR review passes; control access to these review passes by defining TAR security as explained in step 8 of TAR Project Configuration.

To create batches for TAR review passes:

  1. Get started:

    1. Start Eclipse and log in as an Eclipse administrator.

    2. Open the case for which the TAR training batches are being created.

  2. In the Case View pane, click the Case Administration tab, , then click Review Pass Management.

  3. In the list of review passes, click the needed TAR review pass.

  4. Click to update data in the Review Pass Information area.

  5. Review the number of unbatched documents in the review pass, then click Create Batches.

  6. After batches are created:

    1. Evaluate data in the Review Pass Information area. Click again to ensure data is updated.

    2. Provide instructions to reviewers.
