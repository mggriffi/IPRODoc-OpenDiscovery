Work with Cluster Sets

Clustering Overview

A cluster is a group of conceptually similar documents existing together in a tree-type hierarchy. automatically applies a descriptive title to each document cluster without requiring user knowledge or input within the data set.

Cluster sets are groups of clusters that are visually represented through color coding in the concept wheel shown in the Visual Search dashboard.

Clustering is based on an algorithm that finds the general major groupings and minor groupings of the data and presents cluster results in a balanced hierarchy. The number of cluster levels, size, and other settings can be adjusted before a cluster is created. You may choose to cluster the entire case or a selected saved search.

For more information about the concept wheel, see Define a Visual Search.

Before You Begin

Planning will bring you the greatest success in defining Analytics for each case. To familiarize yourself with required information and/or available options, “walk through” the Analytics configuration function as follows:

  1. You must understand your case data, case documents, and reviewer tasks. For example, review cases and discuss with reviewers the types of clusters and categories that will assist them as they do their jobs.

  2. For categories: If an appropriate analytics index has not been created, complete Create and Manage Search Indexes.

    1. Determine required category sets. Sets are distinguished by the number of allowed documents, similarity score, and/or which search is to be affiliated with the category set. For example, you may have the category set “Energy Issues.”

    2. Determine needed categories, that is, titles that represent the topics users must find in . Categories should be based on specific topics. For example, you may have categories of “Gas,” “Nuclear,” “Solar,” and so on.

    3. Identify examples of content that exemplify the categories you create. Examples may be case documents, document text, and/or other content.

  3. Start and log in as a user who is a member of the Super Admin group, then select each command from the Analytics menu that you will be using. Review the configuration screens and plan your approach.

Create a New Cluster Set

Creating a cluster set in the Analytics screen is similar to how it was done in Administration. For more information, see Define IPRO Analytics Clusters.

The Analytics Settings screen allows you to manage cluster sets.

  1. To create a cluster set, in the left bar click the Analytics Settings icon .The Analytics screen appears.

  2. Click the + to the right of the Cluster Sets pane header. The Create New Cluster Set dialog box appears.

  3. Fill in the fields as appropriate.

    Note: To apply content filters, you must first go into Administration and set up the option. This will create a check box that you can select in the Create New Cluster Set dialog box. For more information, see Create and Manage Content Filters.

  4. Click Create. The Confirm Cluster Creation dialog box appears. The dialog box will inform you whether there are any documents that do not contain extracted text. Those documents will not be included in the cluster set. Click Continue.

  5. Wait for the cluster set to generate. You can either watch the generation process in Job Manager or simply wait for it to finish through the Analytics screen. When it is complete, the cluster set details box will display Complete.

  6. If the cluster set fails to generate, the details box will display Failed. In addition, an error message will appear.

Manage Analytics Permissions

Manage Analytics is the area in the System Manager where you govern how Analytics works. This is done through normal user permissions rather than Super Admin permissions.

  1. Click the Settings icon in the top-right corner of the screen. The Settings icon is a global button that displays in every module of the platform.

  2. The System Manager opens. In the left pane of the System Manager, click Groups.

  3. Click on the name of the group you would like to edit. The group profile opens on the right side of the screen.

  4. Click the Edit icon . You are now able to modify the users, permissions, and cases for the selected group.

  5. On the Permissions tab, expand the Search and Review section, then assign the Manage Analytics permission. To learn more about the available permissions, see Permissions List.

  6. Click Save.
