Export Checklist for Archives

The following steps will guide you through the export creation process. This will help ensure OPEN Discovery productions/exports are ran correctly prior to sending to cloud services.

Create the Export Set

  1. In the Exports Set section for a production select the _ExportTemplate and select the copy icon.

  2. Name the copy of the Export something you’d like to be referenced in the future and also select “Zip upon completion” as this will zip the production up allowing you to provide a single zip of the entire export.

    Note: If you want to retain the original BEGDOC for these documents, skip this next step.

Numbering Options

Numbering Options is where you set the numbering for your documents. For detailed instructions on numbering options, see Select Numbering Options.

Select Output Locations

This is where you will select to include your Natives, Images and Text in the export.

  1. Under the Native section, check the box to include all Natives.

  2. Under the Image section, check the box to include all Images.

    1. Select the Image Type you wish to export. Available Types: Single Page Images (this is the standard), Multi-page images, Single-Page PDFs, Multi-Page PDFs (PDFs can also be made searchable).

  3. Under the Text section, check the box to include Text .

    1. Under File Type, select Document- level text.

The second Output Locations is where you will select the load files you need and the metadata to provide within the load file(s).

  1. Under Load File & Output Locatons, click to open the selection window. Make your selections and click OK.

  2. Under Fields & Tags, click to open the selection window. Select the metadata fields and /or tags and click OK.

Volume Information

The next section is Volume information. This needs to be unique to the exports folder. If this is your first export, you can leave it as it is, but on any future productions, you would want to change this (the easiest way is to increment the starting number by 1).

Image Options

Redaction Options and Endorsement Settings is where you select the options for redactions and the image endorsement.

For detailed instructions on redactions and endorsements, see Select Image Options.

Note: Our default template for an export will not burn any redactions or endorsements.

If you think these documents will need to be produced with a different BEGDOC in the future, it is advised to not do any endorsements as that will result in having the document endorsed with two different numbers.

Run the Export

To validate and run an export job, ensure that all definitions have been completed.

  1. In the Export Tools section of the page, click .

  2. In the Select Documents dialog box, select your saved search and optionally select fields to sort by and their sort order. Click OK.

  3. Click and evaluate the Validation Report. A typical report is shown in the following figure; different entries will exist depending on your export selections.

  1. If the report identifies conflicts, resolve the conflicts. Override indicators are as follows:

    2. Y = Can be overridden

    S = Can sometimes be overridden (depending on circumstances)

    N = Cannot be overridden

  2. If conflicts were found, make needed corrections and/or Override the conflicts as required. If the override button is greyed out, you have something in either your search, or settings that makes the production impossible to run.

  3. After all conflicts are resolved and/or overridden, click to start the export (The button is available only when all conflicts have been addressed.)

  4. Optional: Click (located in the bottom right of the Status area) to refresh the data in the Status area.

  5. Observe Status below Summary in the right pane. Status will change as process steps complete, and may include the terms listed in the following table.

    6. Export Status Details

    Status

    Description

    Not Started

    The export process has not yet begun.

    Staging

    The export data is being placed in the staging database.

    Staging Complete

    All export data is in the staging database.

    Pending

    The export job has been placed in a queue.

    In Progress

    The export data is being processed.

    Paused

    The export job has been placed on hold.

    Deleting

    The export job has been canceled and is retrieving data about the job, such as errors and information about the status of the job at the point of cancellation.

    Deleted

    Cancellation has been completed.

    Complete

    The production job has finished; production data may be transferring from the staging database back to or the data transfer may have finished.

  6. When the export job is complete, export files are ready for use.
