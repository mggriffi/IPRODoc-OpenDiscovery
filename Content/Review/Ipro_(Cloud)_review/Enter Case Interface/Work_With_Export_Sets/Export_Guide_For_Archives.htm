Export Guide for Archives

Use the following guide to help ensure OPEN Discovery productions/exports are ran correctly prior to sending to cloud services.

Production/Export Checklist

Questions To Answer Prior To Running a Production

  1. What is the Saved search that will be used for this production and the expected document count?

  2. Will the documents be re-numbered and if so, what would you like the documents to be numbered: EX: PROD_0000001 or PROD000001.

  3. Image type (Available Types: Single Page Images (this is the standard), Multi-page images, Single-Page PDFs, Multi-Page PDFs (PDFs can also be made searchable).

  4. Natives - default is to only produce the native version of Excel files and other files that cannot be imaged and create a placeholder image for it. Indicate if this will work or if you have a preference.

  5. Text - this would be separate text files - one per document being produced.

  6. Metadata - what columns of data from the database should be included with the production? i.e: email sent/received, email to/from, date created, etc.

  7. Which load files should be included? Typically, we include a DAT file for the metadata, natives & text and either OPT or LFP files for the images. If unsure, and you know what software opposing counsel is using, please contact services and they can provide the proper load files.

  8. Do you need any confidentiality endorsements on the produced documents? If so, what language should be used and where would you like it to appear on the documents?

  9. Other endorsements: Are the documents being endorsed with the bates number?

Note: If you have an agreed upon ESI protocol or a document that discusses the format of production and answers the above questions, we recommend using that as it will have exactly what should be provided.

Once the above information is provided, the production/export can now be started.

Run a Saved Search

  1. Run same search with family if the original search wasn’t bringing in family. If there is a change in the number of documents, please reach out to the client and confirm if this is intentional.

  2. If client mentioned tag(s) they want to include/exclude, perform a tally on the tag group(s) and ensure that proper tags are included/excluded. If there is a discrepancy, decide if they need to be excluded or included.

  3. If the above is accurate to the clients request, select a document from the grid, change the coding form to Productions. Right-click on the production Tag Group > Add Tag and create a tag for this production. Some examples of the tag name could be the name of the production provided by the client, or if nothing is provided, the date the production is being ran EX: 20230215_PROD001

  4. Mass tag all documents in the search with the newly created tag.

  5. Create a new Saved search folder called “PRODUCTIONS”

  6. Create a saved search for the tagged documents

  7. Modify the search to include a new field search variable: “System Has Image” > Not Equal > “Yes” (Ensure that you are not including family in this search)

  8. Send the results through Mass Action Imaging, providing File Name for “Original File name".

  9. You can check the quality of the images sent by using “Admin > “QC - Imaging Searches” > “Imaging Errors/Exceptions

  10. If any documents are returned, please reach out to the Services team so that they can run Imaging QC and resolve those errors.

  11. Verify that images are correct in the case by adding the following to your search: “System Imaging Errors” > Set and make the necessary adjustments

  12. Locate all placeholder documents by modifying the saved search , adding the following: "System Imaging Error Flags">Contains>*place*

  13. Select a document from the grid, change the coding form to Productions. Right-click on the production Tag Group > Add Tag. Name the tag “Native_Placeholder”

  14. On the grid, Mass tag all documents in the search with the newly created tag.

  15. Create production/export based on the information provided by the above checklist

    Export Sets

  17. In the Exports Set section for a production select the _ProductionTemplate and select the copy icon

  18. Name the copy of the production something you’d like to be referenced in the future under the Production tab and also select “Zip upon completion” as this will zip the production up allowing you to provide a single zip of the entire production to opposing counsel.

    • If you want to retain the original begdoc for these documents, skip this next step

  19. Under numbering options, this is where you set the numbering for your documents,

  20. • Under the image section, select the image type you wish to export:

     

     

     

     

  21. In the “Output locations” section you will select 'SPECIFY BY FILE TYPE OR TAG” under the native section

  22. Select “Document Tag” from the drop down, then select your Native placeholder tag. This will tell the system to provide a native of the documents that have a placeholder image.

  23. Click ok when complete.

  24. Under the image section, select the image type you wish to export.

  25. In the next section is where you select the load files you need and the metadata you need to provide within the load file(s).

  26. The next section is volume information, this needs to be unique to the productions folder, if this is your first production, you can leave it as it is, but on any future productions, you would want to change this (the easiest way is to increment the starting number by 1).

  27. The last section is where you select the options for redactions and the image endorsement. Our default template will burn all default redactions, endorse all documents with the image key (that you set in the first section) and endorse any document tagged with a confidentiality tag.

  28. Once completed select save

  29. On the right hand side select " Select Documents" and select your search and set your search options.

  30. Select Validate (You may get items in the validation that can be overridden (by clicking the Override button). If the override button is greyed out, you have something in either your search, or settings that makes the production impossible to run

  31. Assuming everything is correct, click override

  32. click “Export” on the right-hand side

  33. There is a refresh at the bottom right where you can check the progress
