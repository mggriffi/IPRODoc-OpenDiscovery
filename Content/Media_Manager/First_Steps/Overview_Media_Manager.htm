Overview: Media Manager

The Media Manager module provides a solution for efficient, secure media management and processing. Administrators and users tasked with overseeing case media—from initial intake through final processing—can use Media Manager to track and process that data. These users should be familiar with your organization’s media handling processes and the cases associated with the media.

Media Manager eliminates tedious and manual processes associated with identifying, processing, and tracking your media inventory, and helps protect the chain of custody for all case media. You can copy data from media and send it through the processing, filtering, and review phases of the workflow, as well as monitor the status of processing jobs created in Media Manager.

For a quick overview, see the below video.

Media Manager allows you to:

  • Quantify the scope of a data collection

  • Manage data copying from multiple sources and locations

  • Apply “set-it and forget-it” templates to reduce inconsistencies

  • Build a full chain of custody

  • Ensure identification and verification of hard drives and other source types

  • Barcode media for identification, storage & tracking

About Media Manager and Self Service

offers two processing modules: Self-Service and Media Manager. Self-Service allows you to upload files from anywhere, but requires that your data be stored in a container file (see Overview: Self-Service for details). Media Manager requires you to have your data stored in a UNC path location, but provides more customization for your uploads, and will accept non-container files. While Self Service provides an accessible solution for users with limited processing experience, Media Manager is geared towards users who are well-versed in processing and familiar with the other components in use at their site. Self-Service and Media Manager are each separate, stand-alone processing solutions, but files that are uploaded through Self-Service can then be monitored in Media Manager.

About Media and File Types

Any type of electronic media can be managed in Media Manager (hard drive, DVD, network share, etc.), and media contents can include any type of individual files and/or container files, such as archive files (for example, .ZIP files) or forensic image files (for example, .E01 files) in any directory structure.

Acceptable file types include:

  • Images: .GIF, .JPG, and/or .PNG – no file size limit exists.

  • Attachments: .CSV, .DOC/.DOCX, .GIF, .JPG, .PDF, .PNG, .PPT/.PPTX, .TIF, .TXT, .XLS/.XLSX, .ZIP – file size cannot exceed 4 MB.
