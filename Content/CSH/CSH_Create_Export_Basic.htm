Create Export Basics

An export set is a group of documents and/or case data typically intended to be imported into another application, while a production set is a group of documents to be provided to opposing counsel. Both of these data sets are referred to as exports.

The options for export sets, including file type, numbering, output and endorsement choices are outlined below. Output options are based on the files defined in your database (that is, if you do not have native files in your case, none can be exported). An export set must be created prior to running the production.

  1. Click to create a new export.

  2. Complete basic export set information shown in the table below.

    Option

    Description

    Name

    Enter a name for the set.

    Date

    Enter the date the export (or production) job is estimated to be completed or delivered, or click  and select the date.Use a date format of MM/DD/YYYY. A single digit for month and/or day is acceptable.

    Output Location

    Enter the full path for exported files, or click Browse and navigate to/select the needed folder. Make sure the location is in UNC format and the destination folder exists. The path must not already have exported data in it.

    Export Environment

    Click in the Export Environment field and select the environment for this export/production job.

    Notes

    Optional: Enter additional information about this export set.

  3. Continue defining your export with the additional options.

    Note: If you are not ready to define the following options, you can click Save and return at a later time to edit the export set. However, once an export has been run it cannot be modified.
