Overview: Reports

Below is a quick definition of each report available for Processing, Review and Systems listed in Alphabetical order.

Processing Reports

Data Extract De-duplication Summary Report

This report provides a summary of items not processed due to de-duplication settings.

Data Extract Detailed Error Report

This report provides a list of extraction and OCR errors and status messages encountered during data extraction. The end of the report displays a list of password-protected files that includes the Item ID, Location, item type, file, and physical location for each file.

Data Extract Error Summary Report

This report provides a summary of errors encountered during processing.

Data Extract Filter Manager Summary Report

This report provides results of rules applied and detail on how they affected the final total of promoted documents.

Data Extract Item Summary

Summary of items processed by type and category.

Discovery Detailed Error Report

This report provides a list of extraction and OCR errors and status messages encountered during discovery. The end of the report displays a list of password-protected files that includes the Item ID, Location, item type, file, and physical location for each file.

Discovery Summary Report

Summary of items discovered by item type and category.

Export Series Summary Report

This report provides results of all exports within a series based on Export Jobs and their associated Volumes. Additional information is provided that breaks down the exported documents by sizes and type. Files not exported are also broken down by type along with any error information associated with exports within the series.

Language Summary Report

This report provides a breakdown of identified languages grouped by file type and further by language.

Processing Case Storage Report

This report presents information on the processed data for a case over a specified date range, aggregated at the project, custodian, job type, and individual job level. Date range is optional. Columns may be sorted. All sizes are exported in GB.

Source Container Report

This report provides a list of the top level containers for selected case

Reviewer Reports

Case Data Summary Report

This report summarizes case size details such as number and total size (in GB) of files (image, native, and text files). If documents have been part of a production set, production size details also included. Report can be based on any field defined in Eclipse with the Category option. For example, if a CUSTODIAN field is defined with the Category option, a report could be generated to summarize size details for all documents broken down by custodian.

Gap Report

Identifies missing pages, that is, breaks in the numbering sequence of image keys (case, production, or other document numbers) for a particular case. For example, if ECL000009 is a single-page document and the next document number in the case is ECL000012, a gap exists and will be included in the report.

Privileged Report

This report presents documents based on tags selected for the report. Included are document totals, the applied tag(s), and field data and other applied tags as defined by your administrator.

Production Deliverable Report

This report presents details on production and imported production history jobs.

Production Summary Report

This report summarizes production and imported production history jobs completed in Eclipse Administration.

Redacted Documents Report

This report summarizes redaction activity for a selected case.

Redaction Detail Report

This report includes detailed information on redacted documents as defined by your administrator.

Review Case Storage Report

This report returns the size of all records in the associated case database that were imported within the selected date range. End Date represents the most recent date that terminates the desired date range. All parameters are optional.

Review Status by Custodian Report

This report presents review status of documents in a particular review pass, sorted by a field selected by your administrator that contains custodian details. Tag details may also be included if configured by your administrator.

Review Status by Reviewer Report

This report includes review status of documents in a particular batch, sorted by assigned reviewers. Tag details may also be included if configured by your administrator.

Reviewer Edits Per Day Report

This report lists all reviewers by user name and the number of distinct actions (field editing and/or tagging) each reviewer has made on each day of the date range, for a specific date range. Can be useful in evaluating review efficiency.

Reviewer History Report

This report summarizes reviewer activities (such as field editing and tagging) for a selected period of time, including total hours spent by each reviewer, number of edits, and edits per hour and day.

Reviewer History Details Report

This report provides detailed information on reviewer activities (such as field editing and tagging) for a selected period of time, including day-by-day summary of time spent and editing details.

Reviewer Batch History Details Report

This report provides detailed information on reviewer activities in a given batch, including check out and check in times, view counts, and editing details.

Summary by Custodian Report

This report summarizes review activities (number of documents reviewed, not reviewed, etc.) sorted by a field selected by your administrator that contains custodian details.

TAR Project Certification Details Report

This report presents detailed results for the certification round, including precision data for the review in terms of how accurately non-responsive documents have been identified by the TAR process.

TAR Project Discrepancies Report

This report identifies documents for which there is a disagreement between the reviewer and the TAR analysis regarding document responsiveness.

TAR Project Score Distribution Report

This report presents distributed category scores (per validation round) for all rounds in the TAR project.

TAR Project Summary Report

Provides an overview of the TAR project, with data ranging from the total number of documents in the TAR project (based on the category index selected for the project), to projected Responsive and Non Responsive documents for the project.

TAR Validation Details Report

Presents detailed results for each validation round, including precision data for the review in terms of how accurately Responsive documents have been identified by the TAR process.

System Reports

File Type Report

This report summarizes media file count and file size based on file extensions.

IPRO Processing Storage Report

Reports system-level aggregations of processed data over a specified date range, as well as current database sizes. All parameters are optional. Columns may be sorted. All sizes are exported in GB. Default granularity is set to Managing Client / Client Level.

IPRO Review Storage Report

Reports the total size of all records in the selected Managing Client's case database(s) that were loaded within the selected date range. End Date will represent the most recent date that terminates the desired date range. Default aggregation is by Case, Client, and Managing Client. All parameters are optional.

User Billing Report

This report provides User Name, Days Active, Managing Client, and Date user was created/enabled in Eclipse for a selected time period.
