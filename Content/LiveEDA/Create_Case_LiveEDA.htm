Create LIVE EDA Case

A (Early Data Assessment) case created in allows you to perform searches on live content across multiple repositories and promote the relevant data to your  Review case.

Note: To create an Early Data Assessment case in , you must configure LIVE EDA in both and the , as well as configure Indexes and Archives. For more information, see Configure LIVE EDA.

In the Case Management module, you can create the different case types - a case, Processing case, and Review case - simultaneously, so that you can ensure that they are interconnected. This topic explains how to create an case. If you are creating the case to automatically load data for processing, you can include a Processing case. If you would like to promote the data to Review, you can include a Review case. For more information about cases, see Overview: Case Management and Create a New Case in OPEN DISCOVERY.

Complete the steps outlined below to create a new case in .

Note that in the configuration pages, required entries are marked with a red asterisk, *.

  1. Open the Case Management module.

  2. Define the managing client and client. Take one of the following actions:

    • If the managing client and/or client do not exist, complete the steps listed in Create Managing Client and Client.

    • If the managing client and/or client exist, navigate to the relevant tile.

  3. Determine whether an existing or a new code name is required for the new  case. Take one of the following actions:

    • To create a new code name for the new case, click the hamburger icon corresponding to the client. Select Add Case.

    • To create the new case for an existing code name, click the hamburger icon corresponding to the code name. Select Add Case.

  4. On the Create Case page, select Early Data Assessment and click Create.

  5. In the first section of Case Creation, you confirm the Code Name*, Matter Number, and Client. Review the details in the following table.

    ITEM

    Description

    Code Name*

    Enter the new code name by using a maximum of 256 Unicode characters. For example, “Smith v. Jones.”

    Matter Number

    Optional: Enter a matter number by using a maximum of 128 Unicode characters.

    Client

    Review the selected client. If the wrong client was selected, click Cancel and start again with the correct client.

  6. Take one of the following actions:

    • If creating a new code name for the new case, define and enter the new code name in the Code Name* field.

    • If creating the new case for an existing code name, ensure that the code name generated in the Code Name* field is still correct. If not, click Cancel and create a new code.

  1. Click Next.

  2. In the second section of Case Creation, first define the name of the case. Enter the new case name in the Case Name* field.

  3. Specify the relevant Connectors and Custodians*. Your selection determines which live data will be included in the case.

  4. Select the appropriate Connectors*. Perform one of the following actions in the left panel of the Connectors and Custodians field: 

    • To include all available locations, click Select All Locations.

    • To include specific locations only, manually select the relevant connectors from the available list by selecting the corresponding checkboxes.

  5. Select the appropriate Custodians*. Perform one of the following actions in the right panel of the Connectors and Custodians field:

    • To include all associated custodians, enable the Select All option.

    • To include specific custodians only, manually select the relevant custodians from the available list by selecting the corresponding checkboxes. To search for a custodian, type the custodian name in the Search Custodian Name search bar and press enter.

  6. Review the selected Connectors and Custodians*. Make sure that you did not skip any pages.

  7. In the Groups* section, select the group of users granted access to the case. All users in the selected group will have permission to open, view, and perform all of the available actions in the case.

    Note: Groups specify the users granted access permissions. For more information about how to create, make changes to, and manage users and groups, see Manage Users and Groups.

  1. Click Next.

  2. In the third and final section of Case Creation, review the Summary of Case Settings. To make changes or corrections, click Back.

  3. When satisfied, click Create Case. Wait as the case is created. When finished, the case displays in the Case Management page summary.

    Note: To make changes to a case after it has been created, navigate to the Case Overview, click the hamburger icon on the relevant case tile, and select Edit Case. For information about how to edit a case, see Edit LIVE EDA Case Settings.
